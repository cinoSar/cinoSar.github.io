0.1 — The Final Project Goal

This course is designed around one cumulative QGIS project rather than a series of disconnected examples. By the end, you will have built a working GIS project based on real data from Padova, combining geocoded firm records with polygon layers such as zoning and territorial boundaries.

The aim is not simply to learn where QGIS tools are located, but to understand how those tools fit into a research workflow: importing data, checking that it is usable, mapping it clearly, asking location-based questions, and exporting outputs that can support reports, slides, or a website.

By the end of the course, you should be able to:

  • set up a clean QGIS project and keep it organised;
  • load and inspect both tabular and spatial data;
  • style firm points and polygon layers in a readable way;
  • filter, join, and analyse data spatially;
  • create a polished map layout for presentation or publication;
  • automate selected tasks through the QGIS Python Console.

0.2 — The Course Data

The course uses a deliberately small but realistic collection of files so that you can focus on understanding GIS workflows without becoming overwhelmed by too many layers at once.

The main materials are:

  • Padova firm data in tabular form, including geocoded records that can be displayed as points on a map;
  • zoning layers (Zonizzazione Territoriale Omogenea), used to understand the spatial context of firms — residential, commercial, industrial, and agricultural zones;
  • perimeter layers, used for selections, clipping, and introductory spatial analysis;
  • supporting style and metadata files, which help explain how GIS datasets are structured behind the scenes.

These files are suitable for beginners because they allow you to work with both attribute information and spatial geometry from the very beginning.

0.3 — How You Will Learn in This Course

Each lesson follows the same teaching structure so that you always know what to expect.

Guided replication

You first reproduce the workflow step by step inside QGIS. This is where you learn the interface, the sequence of actions, and the expected result.

Checkpoint

At the end of the guided section, you compare your result with the expected outcome before moving on. This prevents small mistakes from accumulating across lessons.

Open-ended task

Each lesson then ends with a related challenge. At the beginning of the course, this task may feel very close to the guided replication. Later on, it will ask you to use a different or more powerful tool to achieve a related result.

Pedagogical idea

The course begins by giving you confidence through replication. It then gradually shifts towards transfer: using similar methods on slightly different problems, and recognising when a stronger QGIS tool is preferable to a basic one.

0.4 — The Python Sidebar

The main path through the course is fully graphical. You do not need to write code in order to complete the lessons or understand the core GIS workflow.

Every lesson includes a Python sidebar that progressively builds your ability to automate GIS tasks using the QGIS Python Console. If you follow the sidebar from the beginning, you will have a working scripted workflow by the end of the course. If you prefer to skip it entirely, the GUI path is self-contained.

For the Python sections, you should already feel comfortable with:

  • variables, lists, dictionaries, and functions;
  • for loops and if statements;
  • imports and simple debugging;
  • basic file paths and folder structures.

Open Task

Your first challenge

Before Lesson 01, write a short note for yourself answering these two questions:

  1. What would you like to be able to do with QGIS by the end of this course?
  2. Which part feels most unfamiliar right now: spatial data, map design, coordinate systems, or analysis?

This small reflection will give you a personal baseline. When you complete the course, you can compare your final skills with the uncertainty you started from.
