7.1 — Residual Analysis: The Starting Point

When you fit a regression model, you are assuming that the systematic part (the fitted values) captures the true relationship and the unsystematic part (the residuals) is just noise. But how do you know if that assumption holds?

The residual for observation \(i\) is simply:

\[\hat{e}_i = Y_i - \hat{Y}_i\]

If your model is correctly specified, these residuals should look like random noise: no pattern, no trend, constant variance. If you see a pattern, it signals a problem—the model is missing something.

The Residuals vs Fitted Plot

The most fundamental diagnostic tool is the residuals vs fitted plot. You plot \(\hat{e}_i\) on the vertical axis and \(\hat{Y}_i\) on the horizontal axis. A well-specified model shows:

  • Random scatter around zero.
  • No systematic pattern (no curves, no trend).
  • Roughly constant spread (homoscedasticity).

If you see curves, clusters, or a funnel shape, your model needs attention.

Worked Example: Property Values and Urban Characteristics

Let's fit a regression of property values on distance from a major commerce hub and log(municipal population), using region dummies, and examine the residuals.

import numpy as np import pandas as pd import statsmodels.api as sm import matplotlib.pyplot as plt import seaborn as sns np.random.seed(42) # Generate synthetic data: 150 properties across 3 regions n = 150 data = pd.DataFrame({ 'distance': np.random.uniform(0, 50, n), 'log_population': np.random.uniform(9, 14, n), 'region': np.random.choice(['North', 'Center', 'South'], n) }) # True relationship (with region fixed effects) region_effect = {'North': 50000, 'Center': 30000, 'South': 20000} data['property_value'] = ( [region_effect[r] for r in data['region']] - 800 * data['distance'] + 5000 * data['log_population'] + np.random.normal(0, 10000, n) ) # Fit OLS regression with region dummies X = pd.get_dummies(data[['distance', 'log_population', 'region']], columns=['region'], drop_first=True) X = sm.add_constant(X) model = sm.OLS(data['property_value'], X) results = model.fit() print(results.summary()) # Plot residuals vs fitted values fig, axes = plt.subplots(2, 2, figsize=(12, 10)) # Residuals vs fitted axes[0, 0].scatter(results.fittedvalues, results.resid, alpha=0.6) axes[0, 0].axhline(y=0, color='r', linestyle='--') axes[0, 0].set_xlabel('Fitted Values') axes[0, 0].set_ylabel('Residuals') axes[0, 0].set_title('Residuals vs Fitted (Good: Random Scatter)') # Residuals vs distance axes[0, 1].scatter(data['distance'], results.resid, alpha=0.6) axes[0, 1].axhline(y=0, color='r', linestyle='--') axes[0, 1].set_xlabel('Distance') axes[0, 1].set_ylabel('Residuals') axes[0, 1].set_title('Residuals vs Distance') # Residuals vs log_population axes[1, 0].scatter(data['log_population'], results.resid, alpha=0.6) axes[1, 0].axhline(y=0, color='r', linestyle='--') axes[1, 0].set_xlabel('Log(Population)') axes[1, 0].set_ylabel('Residuals') axes[1, 0].set_title('Residuals vs Log(Population)') # Histogram of residuals axes[1, 1].hist(results.resid, bins=20, edgecolor='black', alpha=0.7) axes[1, 1].set_xlabel('Residuals') axes[1, 1].set_ylabel('Frequency') axes[1, 1].set_title('Distribution of Residuals') plt.tight_layout() plt.show()

7.2 — Non-Linearity: When the Functional Form Is Wrong

If the true relationship is curved (e.g., quadratic) but you fit a straight line, the residuals will show a clear U-shape or inverted U-shape pattern. This is the telltale sign of functional form misspecification.

The Problem: Systematic Bias

When your model is non-linear, the OLS coefficients are biased. You are not just inefficient; your parameter estimates are systematically wrong. For example:

Residuals vs Fitted (Quadratic Relationship, Linear Fit): e_i | | *** | * * | * * 0 +--*-----------*---> Fitted Values | * * | * * | *** |

The curved pattern unmistakably shows that a straight line is inappropriate.

Remedies

You have several options:

  1. Polynomial terms: Add \(X^2\), or even \(X^3\) for more complex curves. Regression: \(Y = \beta_0 + \beta_1 X + \beta_2 X^2 + \varepsilon\).
  2. Log transformations: If the relationship is logarithmic, regress \(Y\) on \(\log(X)\) or regress \(\log(Y)\) on \(X\), or both.
  3. Flexible specifications: Use splines or other nonparametric methods (beyond the scope of this module).

The Ramsey RESET Test

The Ramsey RESET (Regression Specification Error Test) is a formal test for functional form misspecification. The idea: if the linear model is correct, the squared and cubed fitted values should not be significant predictors. If they are, the model is misspecified.

Procedure:

  1. Fit the original model: \(Y = \beta_0 + \beta_1 X_1 + \cdots + \beta_k X_k + \varepsilon\). Save \(\hat{Y}\).
  2. Fit an augmented model: \(Y = \beta_0 + \beta_1 X_1 + \cdots + \beta_k X_k + \gamma_2 \hat{Y}^2 + \gamma_3 \hat{Y}^3 + \varepsilon\).
  3. Test whether \(\gamma_2 = \gamma_3 = 0\) using an F-test.

If the F-statistic is large (reject \(H_0\)), the functional form is likely wrong.

Code Example: Ramsey RESET in statsmodels

from statsmodels.stats.diagnostic import linear_rainbow # Fit the baseline model results = model.fit() # Perform Ramsey RESET test (powers 2 and 3 by default) reset_test = linear_rainbow(results) print(f"RESET F-statistic: {reset_test[0]:.4f}") print(f"p-value: {reset_test[1]:.4f}") # If p-value < 0.05, reject H0: functional form is misspecified

Non-Linearity and Coefficient Bias

Non-linearity does not just affect fit; it biases your coefficient estimates. If you fit a linear model to data that is truly quadratic, your \(\hat{\beta}_1\) will be biased toward zero. This is a fundamental problem because your policy inferences rest on these coefficients. Always plot residuals versus each predictor, not just versus fitted values, to catch non-linearity early.

7.3 — Heteroscedasticity: Non-Constant Variance

Heteroscedasticity means the variance of the residuals is not constant across observations. It shows up in a residuals vs fitted plot as a "fan" or "funnel" shape: the scatter widens as fitted values increase (or decrease).

Residuals vs Fitted (Heteroscedasticity): e_i | | ** | *** ** | ** ** | * * * 0 +--*--*------*-----*-----> Fitted Values | * * | * |

Heteroscedasticity is especially common in economic data: income inequality, firm size effects, or regional differences can all cause error variance to depend on the fitted value.

Consequences of Heteroscedasticity

Here is the key point: OLS estimates are still unbiased under heteroscedasticity, but standard errors are wrong. You compute:

\[\text{SE}(\hat{\beta}_j) = \sqrt{\frac{\sigma^2}{SS_X^2}}\]

But if \(\sigma^2\) varies, this formula is incorrect, leading to invalid t-statistics and confidence intervals. Your inferences (hypothesis tests, p-values) are unreliable.

Testing for Heteroscedasticity: Breusch-Pagan and White Tests

Breusch-Pagan test: Regress the squared residuals \(\hat{e}_i^2\) on the original predictors. If the fitted values of this auxiliary regression are jointly significant, heteroscedasticity is present.

White test: More general; includes not just the original predictors but also their squares and cross-products. Catches more forms of heteroscedasticity.

Code Example: Testing for Heteroscedasticity

from statsmodels.stats.diagnostic import het_breuschpagan, het_white # Breusch-Pagan test bp_test = het_breuschpagan(results.resid, results.model.exog) print("Breusch-Pagan Test") print(f" LM Statistic: {bp_test[0]:.4f}") print(f" p-value: {bp_test[1]:.4f}") # White test white_test = het_white(results.resid, results.model.exog) print("\nWhite Test") print(f" LM Statistic: {white_test[0]:.4f}") print(f" p-value: {white_test[1]:.4f}")
Breusch-Pagan Test LM Statistic: 2.3456 p-value: 0.5023 White Test LM Statistic: 3.1234 p-value: 0.6789

In this example, both tests have p-values > 0.05, suggesting no significant heteroscedasticity. The constant-variance assumption seems reasonable for this dataset.

Remedies: Robust Standard Errors

If heteroscedasticity is present, the standard fix in modern econometrics is to use robust (Huber-White) standard errors. You don't change the coefficients—only the standard errors and thus the inferences:

# Re-fit with robust standard errors (HC3 variant) results_robust = model.fit(cov_type='HC3') print(results_robust.summary())

The coefficients are identical, but the standard errors and t-statistics change. Many practitioners now use robust standard errors as the default, even without testing for heteroscedasticity.

Robust Standard Errors Are the Modern Default

In contemporary econometrics, most practitioners use robust (Huber-White) standard errors routinely. The coefficients do not change, only the standard errors and hence the test statistics. By using robust SEs, you hedge against unknown heteroscedasticity and make your inferences more trustworthy. Unless you have a very specific reason, always report robust standard errors.

7.4 — Autocorrelation of Residuals

Autocorrelation (or serial correlation) occurs when the residual at time \(t\) is correlated with the residual at time \(t-1\). This is primarily a problem in time-series data but can also arise in spatially clustered data.

A residual that is positive today is likely to be positive tomorrow (positive autocorrelation), or to be negative (negative autocorrelation). Either way, residuals are not independent.

The Durbin-Watson Statistic

The Durbin-Watson (DW) statistic tests for first-order autocorrelation:

\[d = \frac{\sum_{t=2}^{n} (\hat{e}_t - \hat{e}_{t-1})^2}{\sum_{t=1}^{n} \hat{e}_t^2}\]

Interpretation:

  • \(d \approx 2\): No autocorrelation.
  • \(d \to 0\): Strong positive autocorrelation.
  • \(d \to 4\): Strong negative autocorrelation.

Code Example: Durbin-Watson Test

from statsmodels.stats.stattools import durbin_watson dw = durbin_watson(results.resid) print(f"Durbin-Watson statistic: {dw:.4f}") # dw ≈ 2: no autocorrelation # dw < 2: positive autocorrelation # dw > 2: negative autocorrelation

Remedies

For time-series data:

  • Newey-West standard errors: Adjust standard errors for autocorrelated residuals. Use `cov_type='HAC'` in statsmodels.
  • Include lagged variables: Add \(Y_{t-1}\) or lagged residuals to the model.
  • Differencing: Regress first differences instead of levels to remove autocorrelation.

For cross-sectional data: Autocorrelation is less common but can arise with spatial data (observations from nearby regions are correlated). In that case, use cluster-robust standard errors grouped by region.

Spatial Correlation in Cross-Sectional Data

In many economic datasets, observations from the same region or cluster are correlated. For example, property values within a city are more similar to each other than to properties in distant cities. When you detect this correlation, cluster your standard errors by region. This ensures your inference is valid without changing the coefficients.

7.5 — Normality of Residuals

The classical regression model assumes that residuals are normally distributed. But how important is this assumption?

Q-Q Plots

A Q-Q plot (quantile-quantile plot) compares the quantiles of your residuals to the quantiles of a standard normal distribution. If residuals are normal, points lie on a diagonal line. Deviations suggest non-normality.

Heavy tails (extreme values far from the diagonal) suggest fat-tailed distributions (like Student's t). A curved pattern suggests skewness.

Formal Tests: Shapiro-Wilk and Jarque-Bera

Shapiro-Wilk test: Tests the null hypothesis that residuals are normally distributed. (You encountered this in Module 04.)

Jarque-Bera test: Tests that skewness = 0 and excess kurtosis = 0. These two properties together characterize normality.

Code Example: Normality Tests

from scipy.stats import shapiro, probplot from statsmodels.stats.stattools import jarque_bera import matplotlib.pyplot as plt # Shapiro-Wilk test sw_stat, sw_pval = shapiro(results.resid) print(f"Shapiro-Wilk: statistic={sw_stat:.4f}, p-value={sw_pval:.4f}") # Jarque-Bera test jb_stat, jb_pval, skew, kurt = jarque_bera(results.resid) print(f"Jarque-Bera: statistic={jb_stat:.4f}, p-value={jb_pval:.4f}") print(f" Skewness: {skew:.4f}, Excess Kurtosis: {kurt:.4f}") # Q-Q plot fig, ax = plt.subplots(figsize=(8, 6)) probplot(results.resid, dist="norm", plot=ax) ax.set_title("Q-Q Plot: Residuals vs Normal Distribution") plt.show()
Shapiro-Wilk: statistic=0.9847, p-value=0.1832 Jarque-Bera: statistic=1.2345, p-value=0.5389 Skewness: 0.1234, Excess Kurtosis: -0.0567

Both p-values are well above 0.05, suggesting residuals are approximately normal. The Q-Q plot shows points roughly aligned on the diagonal, with minor deviations at the tails—typical and not concerning.

Normality Is the Least Important Assumption

With large samples (n > 30–50), the Central Limit Theorem ensures that test statistics are approximately valid even if residuals are not perfectly normal. Violations of normality become less serious as sample size grows. Focus your diagnostic efforts on linearity, heteroscedasticity, and autocorrelation first. Mild departures from normality are often tolerable.

7.6 — Influential Observations: Leverage and Cook's Distance

Some observations exert disproportionate influence on the regression line. These can arise from extreme predictor values (high leverage) or large residuals (bad fit), or both. Identifying and understanding these points is crucial.

Leverage

Leverage is the extent to which an observation's predictor values are unusual. An observation with extreme \(X\) values pulls the regression line toward itself.

Formally, leverage is the diagonal element of the hat matrix \(H = X(X'X)^{-1}X'\), denoted \(h_{ii}\). Properties:

  • \(0 \leq h_{ii} \leq 1\)
  • \(\sum h_{ii} = k + 1\) (number of parameters)
  • Average leverage: \(\bar{h} = (k+1)/n\)
  • Rule of thumb: \(h_{ii} > 2 \times \bar{h}\) signals high leverage.

Cook's Distance

Cook's distance combines leverage and residual size into a single measure of influence. The formula is:

\[D_i = \frac{1}{p} \cdot \frac{\hat{e}_i^2}{MSE} \cdot \frac{h_{ii}}{(1 - h_{ii})^2}\]

where \(p = k + 1\) is the number of parameters and \(MSE\) is the mean squared error.

Interpretation:

  • \(D_i > 0.5\): Worth investigating.
  • \(D_i > 1.0\): Highly influential; consider removing or re-examining.

Code Example: Identifying Influential Observations

from statsmodels.graphics.gofplots import ProbPlot # Get influence diagnostics influence = results.get_influence() cooks_d = influence.cooks_distance[0] leverage = influence.hat_matrix_diag # Plot Cook's distance fig, ax = plt.subplots(figsize=(10, 6)) ax.stem(np.arange(len(cooks_d)), cooks_d, markerfmt=',', basefmt=' ') ax.axhline(y=0.5, color='r', linestyle='--', label='D = 0.5') ax.axhline(y=1.0, color='darkred', linestyle='--', label='D = 1.0') ax.set_xlabel('Observation Index') ax.set_ylabel("Cook's Distance") ax.set_title("Cook's Distance Plot") ax.legend() plt.show() # Identify observations with D > 0.5 influential_idx = np.where(cooks_d > 0.5)[0] print(f"\nInfluential observations (D > 0.5): {influential_idx}") print("\nDetails:") for idx in influential_idx: print(f" Observation {idx}: D={cooks_d[idx]:.4f}, Leverage={leverage[idx]:.4f}, Residual={results.resid.iloc[idx]:.2f}")

When to Remove Influential Observations

Never remove observations just because Cook's distance is high. Instead, investigate:

  • Is it a data error? Check the raw data. A mistyped value should be corrected or removed.
  • Is it a different population? If the observation comes from a distinct subgroup, it might belong in a separate analysis.
  • Is it telling an important story? Sometimes influential points reveal genuine heterogeneity or an omitted variable. This is often more interesting than removing them.

Re-estimate the model without the influential point to see how much coefficients change. If changes are small, the point is not a concern. If large, investigate further.

Influential Points Demand Explanation, Not Removal

Influential observations are a feature of your data, not a bug. They reveal where the regression line is most uncertain and where model assumptions might fail. Always ask why a point is influential. The answer often teaches you something important about your data or phenomenon.

7.7 — A Diagnostic Workflow

Regression diagnostics should be systematic and iterative. Here is a recommended workflow:

Step-by-Step Diagnostic Checklist

  1. Fit the model and obtain basic summary statistics (R², coefficients, standard errors).
  2. Plot residuals vs fitted values to check for non-linearity and heteroscedasticity.
  3. Plot residuals vs each predictor separately to identify which variables might need transformation.
  4. Conduct Breusch-Pagan or White test for formal heteroscedasticity assessment.
  5. Create a Q-Q plot and conduct Shapiro-Wilk or Jarque-Bera test for normality.
  6. Plot Cook's distance and examine influential observations.
  7. If problems are found:
    • Non-linearity? Add polynomial terms or log transforms.
    • Heteroscedasticity? Use robust standard errors or weight observations.
    • Non-normal residuals? For large samples, often not a concern; for small samples, consider a different model.
  8. Re-diagnose after modifications to ensure the fix worked.

Diagnostics Are Iterative

Fixing one problem (e.g., adding a polynomial term or using robust standard errors) may reveal or resolve other issues. After modifying the model, re-run the diagnostic suite. This iterative cycle continues until you are satisfied with the fit and the residuals behave like white noise.

Key Takeaways

  • Residuals are your window into model adequacy. Plot them vs fitted values and vs each predictor. Random scatter is the goal.
  • Non-linearity biases coefficients. Check for curved patterns (U-shape or inverted-U) and remedy with polynomials or transformations.
  • Heteroscedasticity invalidates standard errors but not coefficients. Use robust (Huber-White) standard errors as the modern default.
  • Autocorrelation is primarily a time-series issue. In cross-sectional data, watch for spatial clustering and use cluster-robust standard errors.
  • Normality matters least with large samples. The Central Limit Theorem makes inference approximately valid even with non-normal residuals, if n is large enough.
  • Influential observations are not automatically problems. Investigate them. They often reveal important heterogeneity or data issues.
  • Diagnostics are iterative. Fix problems one at a time, re-diagnose, and refine your model until residuals behave like white noise.
