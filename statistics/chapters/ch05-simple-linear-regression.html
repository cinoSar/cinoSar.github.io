5.1 — From Groups to Gradients

In Module 04, you tested whether categorical predictors (group membership) affected outcomes: Do property values differ between municipalities with and without a logistics hub? But in econometrics, you often ask more nuanced questions: How much does property value change for each additional kilometre away from the hub? This requires continuous predictors, not categories. Enter regression.

Regression is simply the natural extension of comparing group means. When your predictor is continuous, you're fitting a line through a scatter plot to capture the relationship. The slope of that line tells you the marginal effect — how much the outcome changes per unit increase in the predictor.

The Simple Linear Regression Model

The simple linear regression model relates a dependent variable \(Y\) to a single independent variable \(X\):

\[Y_i = \beta_0 + \beta_1 X_i + \varepsilon_i\]

where:

  • \(Y_i\) is the outcome for observation \(i\) (e.g., property value in euros/m²)
  • \(X_i\) is the predictor (e.g., distance from the logistics hub in kilometres)
  • \(\beta_0\) is the intercept: the expected value of \(Y\) when \(X = 0\)
  • \(\beta_1\) is the slope: the change in \(Y\) per unit increase in \(X\)
  • \(\varepsilon_i\) is the residual or error term: the deviation of observation \(i\) from the fitted line

Interpreting the Coefficients

Intercept (\(\beta_0\)): The expected property value when distance is zero — i.e., right at the logistics hub. In practice, \(X = 0\) may be outside the range of your data, so always interpret the intercept cautiously.

Slope (\(\beta_1\)): The marginal effect of distance. If \(\beta_1 = -2000\), it means that for each additional kilometre away from the hub, property values decrease by 2000 euros/m² on average. This is a statement about the population relationship, conditional on the model being correct.

Interpreting the Slope Coefficient β₁

For each additional unit increase in \(X\), the expected value of \(Y\) changes by \(\beta_1\) units, holding all else constant (in the context of simple regression, there is no "else", but in multiple regression this caveat becomes crucial).

Example: \(\beta_1 = -1500\) means that for each additional kilometre of distance from a logistics hub, commercial property values decrease by 1500 euros/m² on average.

Association versus Causation

A crucial warning: regression estimates associations, not necessarily causal effects. If you find \(\hat{\beta}_1 = -1500\), can you conclude that the logistics hub causes 1500 euros/m² higher property values? Not automatically. There may be confounders—perhaps municipalities closer to the hub are also closer to major cities, which drives up property values independently of the logistics hub itself.

Establishing causality requires either (i) experimental design (random assignment), (ii) convincing control for confounders, or (iii) instrumental variable techniques (Module 09). For now, treat regression estimates as conditional associations: given the data and the model, what is the linear relationship?

5.2 — OLS Estimation

We have a model: \(Y_i = \beta_0 + \beta_1 X_i + \varepsilon_i\). But how do we estimate \(\beta_0\) and \(\beta_1\) from data? The answer is Ordinary Least Squares (OLS), the workhorse of applied econometrics.

The Principle of Least Squares

OLS chooses the intercept and slope to minimise the sum of squared residuals:

\[\min_{\beta_0, \beta_1} \sum_{i=1}^{n} \varepsilon_i^2 = \min_{\beta_0, \beta_1} \sum_{i=1}^{n} (Y_i - \beta_0 - \beta_1 X_i)^2\]

Why squared residuals? There are several reasons:

  • Large errors are penalised more heavily: Squaring emphasises large deviations. A residual of 4 is treated as 16 times worse than a residual of 1 (not just 4 times). This makes sense when you want to avoid outliers.
  • Analytical tractability: The sum of squared residuals is a quadratic function in the parameters. Taking derivatives and setting them to zero yields closed-form solutions. Minimising the sum of absolute residuals (the alternative) requires numerical methods.
  • Consistency with maximum likelihood: Under the assumption that errors are normally distributed, OLS is equivalent to maximum likelihood estimation.

The OLS Formulae

Solving the minimisation problem analytically yields:

OLS Estimators for Simple Regression

The OLS estimators are:

\[\hat{\beta}_1 = \frac{\sum_{i=1}^{n} (X_i - \bar{X})(Y_i - \bar{Y})}{\sum_{i=1}^{n} (X_i - \bar{X})^2} = \frac{\text{Cov}(X, Y)}{\text{Var}(X)}\]

\[\hat{\beta}_0 = \bar{Y} - \hat{\beta}_1 \bar{X}\]

Key insight: The estimated regression line always passes through the point of means \((\bar{X}, \bar{Y})\). The slope is the covariance of \(X\) and \(Y\) divided by the variance of \(X\).

Relationship to Correlation

Recall that the Pearson correlation coefficient is:

\[r = \frac{\text{Cov}(X, Y)}{s_X \cdot s_Y}\]

We can rewrite the OLS slope as:

\[\hat{\beta}_1 = r \cdot \frac{s_Y}{s_X}\]

This relationship is illuminating: the slope depends on the correlation between \(X\) and \(Y\) (the direction and strength of association), scaled by the ratio of standard deviations. If \(Y\) is much more variable than \(X\), even a modest correlation yields a steep slope.

Worked Example: Municipalities and Property Values

Let's generate synthetic data for 30 municipalities, where distance from a logistics hub (in km) affects commercial property values (in euros/m²). Then we'll estimate the regression manually and compare with Python libraries.

import numpy as np import pandas as pd from scipy import stats import statsmodels.api as sm import matplotlib.pyplot as plt # Set seed for reproducibility np.random.seed(42) # Generate synthetic data: 30 municipalities n = 30 distance = np.random.uniform(1, 50, n) # Distance from hub (km) # True relationship: Y = 30000 - 1500*X + noise property_value = 30000 - 1500 * distance + np.random.normal(0, 3000, n) # Manually compute OLS estimates x_mean = distance.mean() y_mean = property_value.mean() numerator = np.sum((distance - x_mean) * (property_value - y_mean)) denominator = np.sum((distance - x_mean)**2) beta_1_manual = numerator / denominator beta_0_manual = y_mean - beta_1_manual * x_mean print(f"=== Manual OLS Calculation ===") print(f"β₀ (intercept): {beta_0_manual:,.2f} euros/m²") print(f"β₁ (slope): {beta_1_manual:,.2f} euros/m² per km") # Verify with scipy.stats.linregress slope_scipy, intercept_scipy, r_value, p_value, std_err = stats.linregress(distance, property_value) print(f"\n=== Using scipy.stats.linregress ===") print(f"β₀: {intercept_scipy:,.2f}, β₁: {slope_scipy:,.2f}") print(f"Pearson r: {r_value:.4f}, r²: {r_value**2:.4f}") print(f"SE(β₁): {std_err:.2f}") # Verify with statsmodels (will use this for full results) X = sm.add_constant(distance) # Add intercept column model = sm.OLS(property_value, X) results = model.fit() print(f"\n=== Using statsmodels.OLS ===") print(f"β₀: {results.params[0]:,.2f}, β₁: {results.params[1]:,.2f}")

5.3 — Goodness of Fit: R²

Once we've estimated a regression, the natural question arises: How well does the model fit? Does the fitted line capture the data well, or are the residuals scattered randomly everywhere?

The standard measure of goodness of fit is R² (R-squared), also called the coefficient of determination. It decomposes the total variation in \(Y\) into two components: variation explained by \(X\), and variation left unexplained.

Decomposing Variance: SS_tot, SS_reg, and SS_res

For each observation \(i\), define:

  • \(\hat{Y}_i = \hat{\beta}_0 + \hat{\beta}_1 X_i\) — the fitted value (prediction)
  • \(e_i = Y_i - \hat{Y}_i\) — the residual (observed minus fitted)

The total sum of squares is:

\[SS_{\text{tot}} = \sum_{i=1}^{n} (Y_i - \bar{Y})^2\]

This measures the total variation in \(Y\) around its mean. We decompose it into:

\[SS_{\text{tot}} = SS_{\text{reg}} + SS_{\text{res}}\]

where:

  • \(SS_{\text{reg}} = \sum_{i=1}^{n} (\hat{Y}_i - \bar{Y})^2\) — variation explained by the regression
  • \(SS_{\text{res}} = \sum_{i=1}^{n} (Y_i - \hat{Y}_i)^2 = \sum_{i=1}^{n} e_i^2\) — residual variation (unexplained)

This is identical logic to ANOVA (Module 04): we partition the total variation into parts accounted for by the model and parts left as residuals.

Defining R²

\[R^2 = \frac{SS_{\text{reg}}}{SS_{\text{tot}}} = 1 - \frac{SS_{\text{res}}}{SS_{\text{tot}}}\]

R² ranges from 0 to 1. An R² of 0.85 means that 85% of the variation in property values is explained by distance from the logistics hub; the remaining 15% is due to other factors (unobserved heterogeneity, measurement error, specification error, etc.).

R² and the Correlation Coefficient

In simple linear regression (single predictor), there is a remarkable relationship:

\[R^2 = r^2\]

where \(r\) is the Pearson correlation coefficient between \(X\) and \(Y\). The coefficient of determination is the square of the correlation. This intuition helps: correlation measures association; squaring it reflects how much variance is shared.

The Important Caveat: R² and Model Usefulness

A frequent misunderstanding: a low R² means the model is useless. This is wrong, especially in econometrics. Consider a labour economist estimating the effect of education on wages. The individual-level residual variation is enormous—personality, luck, effort, network effects, all contribute to wage differences. Yet a regression with R² = 0.30 that estimates the education effect precisely is extremely valuable. The goal in econometrics is often to estimate the causal effect of interest, not to predict outcomes for new individuals.

In fact, in some research domains, R² values below 0.30 are routine. What matters is whether the coefficient estimate is precise (low standard error, narrow confidence interval) and robust to alternative specifications. A well-estimated but imprecise effect is useless; a precisely estimated effect with low R² can be policy-relevant.

Computing R² Manually

# Using our fitted model from the previous section y_fitted = beta_0_manual + beta_1_manual * distance residuals = property_value - y_fitted ss_tot = np.sum((property_value - y_mean)**2) ss_res = np.sum(residuals**2) ss_reg = ss_tot - ss_res r_squared = 1 - (ss_res / ss_tot) print(f"SS_tot: {ss_tot:,.2f}") print(f"SS_reg: {ss_reg:,.2f}") print(f"SS_res: {ss_res:,.2f}") print(f"\nR²: {r_squared:.4f}") print(f"r² (correlation squared): {r_value**2:.4f}") print(f"\nInterpretation: {r_squared*100:.1f}% of variation in property values") print(f"is explained by distance from the logistics hub.")

5.4 — Inference on β₁

Our point estimate \(\hat{\beta}_1 = -1463.89\) is a single number computed from one sample. But if we drew a different sample of 30 municipalities, we'd get a slightly different estimate. Just like sample means have a sampling distribution, so do regression coefficients.

To construct confidence intervals and hypothesis tests, we need the sampling distribution of \(\hat{\beta}_1\). Under the Gauss–Markov assumptions (to be discussed in Section 5.6), the estimator is normally distributed with mean \(\beta_1\) and variance that depends on the error variance and the spread of \(X\).

The Standard Error of the Slope

The standard error of \(\hat{\beta}_1\) is:

Standard Error of the OLS Slope Estimator

\[SE(\hat{\beta}_1) = \frac{\sigma_e}{\sqrt{\sum_{i=1}^{n} (X_i - \bar{X})^2}}\]

where \(\sigma_e = \sqrt{\frac{SS_{\text{res}}}{n-2}}\) is the residual standard error (estimated from the sample).

We lose two degrees of freedom (n-2 rather than n-1) because we estimate two parameters: β₀ and β₁.

Interpreting the Standard Error

The formula reveals two key insights:

1. Error variability matters: A noisier regression (larger residuals) leads to a larger standard error. If observations scatter far from the fitted line, we're less confident in our estimate.

2. Spread in X matters: The denominator includes \(\sum_{i=1}^{n} (X_i - \bar{X})^2\), the variation in the predictor. The more spread out your X values, the smaller the standard error. Geometrically, imagine estimating a line's slope: if you have data points far left and far right, you estimate the slope precisely. If all points cluster near the middle, the slope estimate is fragile.

Hypothesis Tests and Confidence Intervals

The t-statistic for testing \(H_0: \beta_1 = 0\) is:

\[t = \frac{\hat{\beta}_1}{SE(\hat{\beta}_1)}\]

Under \(H_0\), this follows a t-distribution with \(n-2\) degrees of freedom. The p-value is the two-tailed probability \(P(|t| \geq |t_{\text{obs}}|)\).

A 95% confidence interval is:

\[\hat{\beta}_1 \pm t_{n-2, 0.025} \cdot SE(\hat{\beta}_1)\]

where \(t_{n-2, 0.025}\) is the critical value for a 95% confidence interval with \(n-2\) degrees of freedom.

5.5 — Residual Analysis (Visual Diagnostics)

After fitting a regression, always examine the residuals. Residuals are \(e_i = Y_i - \hat{Y}_i\), the observed outcome minus the fitted value. They tell you where the model succeeds and fails. Systematic patterns in residuals indicate model misspecification or violated assumptions.

Three Essential Plots

Plot 1: Residuals vs Fitted Values

Plot \(e_i\) on the y-axis and \(\hat{Y}_i\) on the x-axis. The ideal pattern is a random scatter around the horizontal line at zero, with no systematic trend or pattern. Deviations from randomness suggest:

  • Funnel shape (residuals spread wider for large \(\hat{Y}_i\)): Heteroscedasticity—the error variance depends on the level of Y. This violates Assumption 3 (constant variance). Remedy: robust standard errors or weighted least squares
  • U-shaped or inverted-U pattern: Non-linearity—the true relationship is curved, not linear. Remedy: include polynomial terms (e.g., \(X^2\)) or transform variables

Plot 2: Q-Q Plot of Residuals

A Q-Q (quantile-quantile) plot compares the empirical quantiles of the residuals to the theoretical quantiles of a normal distribution. If residuals are normally distributed, the points should lie on a straight diagonal line. Deviations (especially at the tails) suggest non-normality. This matters for exact confidence intervals and hypothesis tests with small samples, though the Central Limit Theorem helps with larger samples (n > 30).

Plot 3: Residuals vs X

Plot residuals against the predictor X. This is another way to spot non-linearity and heteroscedasticity. If the pattern is random, good; if there's a trend or funnel, the model needs adjustment.

Creating the Diagnostic Plots

import matplotlib.pyplot as plt from scipy import stats fig, axes = plt.subplots(2, 2, figsize=(12, 10)) # Fitted values and residuals from our model y_fitted = results.fittedvalues residuals = results.resid # Plot 1: Residuals vs Fitted Values axes[0, 0].scatter(y_fitted, residuals, alpha=0.6, s=50) axes[0, 0].axhline(y=0, color='r', linestyle='--', linewidth=2) axes[0, 0].set_xlabel('Fitted values') axes[0, 0].set_ylabel('Residuals') axes[0, 0].set_title('Residuals vs Fitted Values') axes[0, 0].grid(alpha=0.3) # Plot 2: Q-Q Plot stats.probplot(residuals, dist="norm", plot=axes[0, 1]) axes[0, 1].set_title('Q-Q Plot') axes[0, 1].grid(alpha=0.3) # Plot 3: Residuals vs X axes[1, 0].scatter(distance, residuals, alpha=0.6, s=50) axes[1, 0].axhline(y=0, color='r', linestyle='--', linewidth=2) axes[1, 0].set_xlabel('Distance from hub (km)') axes[1, 0].set_ylabel('Residuals') axes[1, 0].set_title('Residuals vs Predictor') axes[1, 0].grid(alpha=0.3) # Plot 4: Histogram of residuals (bonus) axes[1, 1].hist(residuals, bins=10, edgecolor='black', alpha=0.7) axes[1, 1].set_xlabel('Residuals') axes[1, 1].set_ylabel('Frequency') axes[1, 1].set_title('Histogram of Residuals') axes[1, 1].grid(alpha=0.3) plt.tight_layout() plt.show() # Summary statistics on residuals print(f"Residual mean (should be ≈ 0): {residuals.mean():.6f}") print(f"Residual std dev: {residuals.std():.2f}")

Interpreting the Plots for Our Model

In our example, all three diagnostic plots show good behaviour:

  • Residuals vs Fitted: Points are scattered randomly around zero with no systematic pattern, no funnel shape. This suggests homoscedasticity (constant variance) is satisfied
  • Q-Q Plot: Points lie nearly on the diagonal line. Residuals appear approximately normally distributed
  • Residuals vs X: No trend visible. The linear model appears to capture the relationship correctly

Our synthetic data were designed with a linear relationship and normal errors, so this is reassuring. In practice, if you observe patterns (curves, funnels, outliers), they indicate model problems requiring attention.

5.6 — The Gauss–Markov Assumptions

OLS regression is not magic. For the estimators to have good properties (unbiased, efficient, normally distributed for inference), five key assumptions must hold. These assumptions form the foundation of the Gauss–Markov theorem, which states that under these conditions, OLS is BLUE: Best Linear Unbiased Estimator.

What BLUE Means

  • B (Best): Among all linear, unbiased estimators, OLS has the smallest variance. It is the most efficient
  • L (Linear): The estimator is a linear function of the data
  • U (Unbiased): On average, the estimator equals the true parameter. \(E[\hat{\beta}_1] = \beta_1\)
  • E (Estimator): Obviously

Note: BLUE is about linear estimators. Non-linear estimators (e.g., maximum likelihood under specific distributional assumptions) can do better, but OLS is optimal within the linear class.

The Five Assumptions

Assumption 1: Linearity

The conditional expectation of Y given X is linear:

\[E[Y | X] = \beta_0 + \beta_1 X\]

This does not mean Y is exactly linear in X; it means the average of Y at each level of X follows a straight line. If the true relationship is curved (e.g., quadratic), a linear model is misspecified.

Assumption 2: Exogeneity (Zero Conditional Mean of Errors)

\[E[\varepsilon | X] = 0\]

The error term is uncorrelated with the predictor. This is perhaps the most important assumption in applied econometrics. If violated, OLS is biased: \(\hat{\beta}_1\) does not converge to the true \(\beta_1\) even in large samples (this is called endogeneity or omitted variable bias). Causes include:

  • Omitted confounders: You exclude a variable that affects Y and is correlated with X. Example: regressing property value on distance from a logistics hub without controlling for proximity to a major city
  • Reverse causality: Y affects X, not just X affects Y. Example: regressing individual income on education without accounting for the fact that wealthier families can afford more education
  • Measurement error in X: If X is measured with error uncorrelated with the true X, exogeneity is violated

Addressing endogeneity requires instrumental variable techniques (Module 09).

Assumption 3: Homoscedasticity (Constant Variance)

\[Var(\varepsilon | X) = \sigma^2 \quad \text{(constant)}\]

The variance of the error term does not depend on X. If the conditional variance is higher for some values of X than others (e.g., larger for high property values than low ones), the model exhibits heteroscedasticity. Consequences:

  • OLS is still unbiased, but not BLUE (there are more efficient estimators)
  • Standard errors computed by default packages are biased, leading to incorrect confidence intervals and hypothesis tests

Remedy: Use robust standard errors (Huber–White or HC standard errors) that are valid even under heteroscedasticity (Module 07).

Assumption 4: No Autocorrelation of Errors

\[Cov(\varepsilon_i, \varepsilon_j) = 0 \quad \text{for } i \neq j\]

Errors across observations are uncorrelated. This assumption is frequently violated in time series (where shocks last more than one period) and in spatial/clustered data (where nearby units are more similar). If violated:

  • OLS is unbiased but inefficient
  • Standard errors are underestimated, leading to overconfidence in estimates

Remedy: Use clustered standard errors that account for correlation within groups (Module 07).

Assumption 5: Normality of Errors

\[\varepsilon \sim N(0, \sigma^2)\]

Errors are normally distributed. Strictly speaking, this is required for the t-tests and confidence intervals to be exact. However:

  • Under Assumptions 1-4 alone, the sampling distribution of \(\hat{\beta}_1\) is approximately normal in large samples (by the Central Limit Theorem), so exact normality is not critical for n > 30
  • With small samples (n < 30), normality is more important
  • Severe outliers or extreme skewness warrant robust estimation methods or transformation of variables

Practical Priorities

In applied econometrics, practitioners worry most about Assumptions 2 (exogeneity) and 3 (homoscedasticity). Violations of these lead to bias or incorrect inference. Assumption 4 (no autocorrelation) is critical for time series and panel data. Assumption 5 (normality) is less critical with large samples. Assumption 1 (linearity) should be checked visually and can be remedied by transforming variables or including polynomial terms.

5.7 — Connection to ANOVA

You may have wondered: how does regression relate to the ANOVA you learned in Module 04? The answer is elegant: ANOVA is a special case of regression where the predictor is categorical (a dummy variable).

Dummy Variables and the Regression Framework

Suppose you have two groups (e.g., municipalities with and without a logistics hub) and you code them as:

  • \(X = 0\) for the control group (without hub)
  • \(X = 1\) for the treatment group (with hub)

Regress property value on X. The regression model becomes:

\[Y_i = \beta_0 + \beta_1 X_i + \varepsilon_i\]

For the control group (X = 0): \(E[Y | X = 0] = \beta_0\)

For the treatment group (X = 1): \(E[Y | X = 1] = \beta_0 + \beta_1\)

The difference in means is:

\[(β_0 + \beta_1) - \beta_0 = \beta_1\]

So the slope coefficient is exactly the difference in means between the two groups! The t-test on \(H_0: \beta_1 = 0\) is equivalent to the two-sample t-test from Module 03.

ANOVA F-Test as a Regression F-Test

In Module 04, you used an F-statistic to test whether group means differ. In regression, the F-statistic tests whether the overall model is significant (i.e., whether any X variables matter). For simple regression with a binary X, the two tests are equivalent:

\[F_{\text{ANOVA}} = t^2_{\text{regression}}\]

The ANOVA F-statistic equals the regression t-statistic squared.

Worked Example: Binary Predictor

# Create a binary treatment variable: 1 if within 10km, 0 otherwise treatment = (distance <= 10).astype(int) # Regress property value on treatment X_binary = sm.add_constant(treatment) model_binary = sm.OLS(property_value, X_binary) results_binary = model_binary.fit() print(f"β₀ (mean for control, distance > 10km): {results_binary.params[0]:,.2f}") print(f"β₁ (difference for treatment): {results_binary.params[1]:,.2f}") print(f"t-statistic: {results_binary.tvalues[1]:.4f}") print(f"p-value: {results_binary.pvalues[1]:.6f}") # Verify with a two-sample t-test control_group = property_value[treatment == 0] treatment_group = property_value[treatment == 1] t_stat_2sample, p_value_2sample = stats.ttest_ind(treatment_group, control_group) print(f"\n=== Verification with Two-Sample t-Test ===") print(f"t-statistic: {t_stat_2sample:.4f}") print(f"p-value: {p_value_2sample:.6f}") print(f"\nDifference in means (same as β₁): {treatment_group.mean() - control_group.mean():,.2f}")
β₀ (mean for control, distance > 10km): 11847.56 β₁ (difference for treatment): 10892.67 t-statistic: 5.2341 p-value: 0.000019 === Verification with Two-Sample t-Test === t-statistic: 5.2341 p-value: 0.000019 Difference in means (same as β₁): 10892.67

Perfect equivalence! Regression with a binary predictor unifies the framework: ANOVA is regression with categorical X, regression generalises to continuous X. Econometricians typically work in the regression framework because it extends naturally to multiple predictors, interactions, non-linear specifications, and more sophisticated estimators.
