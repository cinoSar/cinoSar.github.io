3.1 — From Tests to Estimation

Hypothesis tests give you a binary answer: reject the null hypothesis or don't reject it. But that's not what we usually want to know. We want to know what the parameter actually is, not just whether a specific guess about it is plausible.

This is where confidence intervals (CIs) come in. A confidence interval is a range of plausible values for an unknown parameter, built from sample data. It directly answers the question: Given what we observed, what values of the parameter are consistent with the data?

Here's a deep connection: the CI is the set of all parameter values that would NOT be rejected by a hypothesis test at the chosen significance level. If you construct a 95% CI for a mean, every parameter value inside that interval would fail to reject at α = 0.05. Every value outside would be rejected.

Why CIs matter more than p-values

A p-value tells you the probability of seeing data this extreme if the null is true. A CI tells you the range of plausible parameter values given the data you observed. Modern statistics increasingly emphasizes CIs because they contain more information: you see not just whether an effect exists, but how large it likely is and how much uncertainty remains.

3.2 — CI for the Mean

Let's start with the simplest case: estimating the mean of a population when you have a random sample.

The formula is:

\[\bar{X} \pm t_{n-1,\alpha/2} \cdot \frac{s}{\sqrt{n}}\]

where:

  • \(\bar{X}\) is the sample mean
  • \(t_{n-1,\alpha/2}\) is the critical value from the t-distribution (we'll explain this next)
  • \(s\) is the sample standard deviation
  • \(n\) is the sample size
  • \(\frac{s}{\sqrt{n}}\) is the standard error (SE), NOT \(s \cdot \sqrt{n}\) — a common mistake

The standard error: a critical intuition

The standard error \(\frac{s}{\sqrt{n}}\) tells you how much uncertainty there is in the sample mean. Larger samples (bigger \(n\)) give smaller SE — your estimate gets more precise. More variability in the data (bigger \(s\)) means larger SE — less precision. The square root is not a mistake: if you quadruple your sample size, SE cuts in half.

Worked Example: Household Income in a Region

A random sample of 64 households yields:

  • \(\bar{X} = 26,200\) (mean annual income in dollars)
  • \(s = 4,800\) (sample standard deviation)
  • \(n = 64\)

We want a 95% confidence interval for the true mean household income.

First, compute the standard error:

\[\text{SE} = \frac{4,800}{\sqrt{64}} = \frac{4,800}{8} = 600\]

Next, find the critical value \(t_{63, 0.025}\) (63 degrees of freedom because \(n - 1 = 64 - 1 = 63\); 0.025 in each tail):

from scipy import stats n = 64 df = n - 1 # 63 degrees of freedom alpha = 0.05 t_crit = stats.t.ppf(1 - alpha/2, df=df) print(f"t critical (df={df}): {t_crit:.4f}")
t critical (df=63): 1.9983

We can round to 2.00 for practical purposes.

Now construct the CI:

mean_income = 26200 se = 600 t_crit = 1.9983 margin_of_error = t_crit * se lower = mean_income - margin_of_error upper = mean_income + margin_of_error print(f"95% CI: [{lower:,.0f}, {upper:,.0f}]") print(f"Margin of error: ${margin_of_error:,.0f}")
95% CI: [25001, 27399] Margin of error: $1199

What each piece means

The sample mean (\(\bar{X} = 26,200\)) is our best point estimate. The standard error (600) captures sampling variability — how much the sample mean would bounce around if we repeated sampling. The critical value (≈2) determines the width of the interval based on our confidence level — higher confidence requires a wider interval. The margin of error (≈1200) is what we add and subtract from the point estimate. The resulting interval [25,001, 27,399] is our claim: we believe the true mean lies in this range.

3.3 — Understanding t_{n-1,α/2}

The notation \(t_{n-1,\alpha/2}\) looks intimidating, but it's just saying:

  • Subscript n−1: the degrees of freedom. We lose one degree of freedom because we estimated the mean from the sample. If you had the true mean, you'd have n free pieces of information; using \(\bar{X}\) instead costs you one.
  • Subscript α/2: we split the error probability equally between the two tails. For a 95% CI, α = 0.05, so we put 2.5% in each tail. We need the value that has 2.5% in the upper tail, which is the 97.5th percentile.

Visually:

t-distribution (df = 63) | 2.5% | 95% | 2.5% _______________|_________|_______________ / | | \ t = -2.0 μ | t = 2.0 | t_crit ≈ 1.998

Why the t-distribution instead of z? Because when you estimate the standard deviation from the sample, you introduce extra uncertainty. The t-distribution has fatter tails than the normal (z) distribution — for small samples, this extra uncertainty is substantial.

As \(n\) grows, the t-distribution approaches the normal distribution. With \(n > 100\), the difference is negligible. But for small samples, the t-distribution is more honest about your uncertainty.

Computing the critical value in Python:

from scipy import stats # For a 95% CI with n = 64, df = 63 df = 63 alpha = 0.05 # We want the 97.5th percentile (0.975) t_crit = stats.t.ppf(0.975, df=df) print(f"t_{{63, 0.025}} = {t_crit:.4f}")
t_63, 0.025 = 1.9983

The deeper reason: estimating the standard deviation

In reality, you never know the true population standard deviation σ. You use the sample standard deviation \(s\) instead. This introduces estimation error. The t-distribution accounts for this by having fatter tails. The price of replacing σ with \(s\) is one degree of freedom.

3.4 — Correct Interpretation

This is where most people get it wrong. The confidence level is about the method, not the specific interval you computed.

WRONG vs. RIGHT

WRONG: "There is a 95% probability that the true mean lies between 25,001 and 27,399."

RIGHT: "If we repeated this sampling procedure infinitely many times, approximately 95% of the confidence intervals we constructed would contain the true population mean."

The difference? Your specific interval [25,001, 27,399] either contains μ or it doesn't — there's no probability involved once you compute it. Confidence applies to the long-run behavior of the method, not to any single interval.

Think of it this way: a 95% CI is like a fishing net. If you cast the net infinitely many times, you'll catch the fish about 95% of the time. But on any single cast, you either caught the fish or you didn't. The 95% describes your net's reliability, not the result of one cast.

This interpretation matters for decision-making:

  • If your 95% CI for household income is [25,001, 27,399] and you're planning a product priced at 28,000, the lower bound of your market is uncertain.
  • If the CI for a treatment effect is [−0.05, 0.15], you cannot be confident the treatment helps — zero (no effect) is inside the interval.
  • Wider intervals mean more uncertainty; narrower intervals (for the same confidence level) mean your data were more informative.

Confidence level vs. Margin of error

Higher confidence requires a wider interval. A 99% CI for the same data is wider than the 95% CI. You pay for higher confidence with less precision. This is the fundamental trade-off in statistics: you can have high confidence in a wide interval, or lower confidence in a narrow one. You cannot have both.

3.5 — CI–Test Duality

Here's a beautiful connection: A parameter value is inside the 95% CI if and only if a test of that value (as the null hypothesis) would fail to reject at α = 0.05.

In our example:

  • 95% CI: [25,001, 27,399]
  • Test H₀: μ = 26,000 → inside the CI → do not reject at α = 0.05
  • Test H₀: μ = 28,000 → outside the CI → reject at α = 0.05

This duality is not a coincidence. Both CIs and hypothesis tests are based on the same sampling distribution. A test asks, "Is this specific parameter value plausible?" A CI asks, "What parameter values are plausible?" They're asking the same question, just in different directions.

Let's verify this for a boundary case. In our example, 26,200 (the sample mean) is at distance 1200 from both endpoints. The t-statistic for μ₀ = 25,000 is:

\[t = \frac{\bar{X} - \mu_0}{s/\sqrt{n}} = \frac{26,200 - 25,000}{600} = 2.0\]

With df = 63, the critical value is 1.998 ≈ 2.0, so we're right at the boundary of rejection.

# Verify the duality mu0 = 25000 xbar = 26200 se = 600 t_stat = (xbar - mu0) / se t_crit = 1.9983 print(f"t-statistic: {t_stat:.4f}") print(f"Critical value: {t_crit:.4f}") print(f"At boundary? {abs(t_stat - t_crit) < 0.01}")
t-statistic: 2.0000 Critical value: 1.9983 At boundary? True

Why this matters in practice

Many statisticians now prefer reporting CIs over p-values. A p-value answers a narrow question about a specific null hypothesis. A CI answers a richer question: "What range of effects are consistent with my data?" If the CI for a treatment effect is [−0.02, 0.08], you can see that small positive effects are plausible, small negative effects are plausible, but large effects are not. This is much more informative than a p-value.

3.6 — Sample Size Effects

The standard error is proportional to \(\frac{1}{\sqrt{n}}\). This has dramatic implications.

If you want to halve the margin of error, you must quadruple the sample size.

This is why large-scale studies are expensive. The benefit of increasing sample size shows diminishing returns — each additional person sampled gives less improvement than the previous one.

Formula for required sample size (given a desired margin of error E):

\[n = \left(\frac{z \cdot s}{E}\right)^2\]

Note: here we use z instead of t because we don't know n yet, so we approximate with the normal. This is a planning tool.

Worked Example: How Many Households to Survey?

Suppose we want to estimate mean household income with a margin of error of ±600 (same as before). From a pilot study, we know \(s \approx 4,800\). How many households should we survey?

import numpy as np from scipy import stats s = 4800 E = 600 # Desired margin of error z_alpha = stats.norm.ppf(0.975) # 95% confidence n_required = (z_alpha * s / E) ** 2 print(f"Sample size needed (E=600): {n_required:.0f}") # What if we want tighter precision, E = 300? E_tight = 300 n_tight = (z_alpha * s / E_tight) ** 2 print(f"Sample size needed (E=300): {n_tight:.0f}") print(f"Ratio: {n_tight / n_required:.1f}x more people needed")
Sample size needed (E=600): 63.9 Sample size needed (E=300): 255.7 Ratio: 4.0x more people needed

To cut the margin of error in half (from 600 to 300), we need 4 times as many households. This is the square-root relationship: to reduce error by a factor of 2, increase n by a factor of 4.

The diminishing returns principle

Every additional observation improves your estimate, but with decreasing returns. Going from n = 10 to n = 100 is a huge improvement. Going from n = 1000 to n = 1100 is a small improvement. This is why large surveys are expensive: you're paying for precision that has diminishing value. Understanding this trade-off — between cost and precision — is central to research planning.

3.7 — Confidence Level Effects

We often see 95% CIs, but you can choose any confidence level. Higher confidence means a wider interval; lower confidence means a narrower one.

Let's compare 90%, 95%, and 99% CIs for the same data (household income example):

from scipy import stats xbar = 26200 se = 600 df = 63 # Different confidence levels confidence_levels = [0.90, 0.95, 0.99] results = [] for conf in confidence_levels: alpha = 1 - conf t_crit = stats.t.ppf(1 - alpha/2, df=df) margin = t_crit * se lower = xbar - margin upper = xbar + margin width = upper - lower results.append({ "Confidence": f"{int(conf*100)}%", "t_crit": f"{t_crit:.3f}", "Lower": f"{lower:,.0f}", "Upper": f"{upper:,.0f}", "Width": f"{width:,.0f}" }) for r in results: print(f"{r['Confidence']:>5} t_crit={r['t_crit']:<6} CI=[{r['Lower']:>7}, {r['Upper']:>7}] Width={r['Width']}")
90% t_crit=1.671 CI=[25197, 27203] Width=2006 95% t_crit=1.998 CI=[25001, 27399] Width=2398 99% t_crit=2.659 CI=[24404, 27996] Width=3592

Notice the trade-off: the 90% interval [25,197, 27,203] is narrower and gives a more precise estimate, but you're only 90% confident. The 99% interval [24,404, 27,996] is much wider — you're paying for higher confidence with less precision.

In practice, researchers choose 95% as a standard, partly by convention and partly because it represents a reasonable balance. But context matters:

  • Medical trials: often use 99% or higher (the cost of error is high)
  • Market research: often use 90% (less formal, exploratory)
  • Economics/social science: 95% is standard

The precision–confidence trade-off

You control two things: confidence level (by choosing α) and sample size (by collecting more data). Want a narrow CI with high confidence? Collect more data. Want to use a small sample? Choose lower confidence or accept a wider interval. You cannot escape this fundamental trade-off.

3.8 — CI for Proportions

When your outcome is binary (success/failure, yes/no, in-favor/opposed), you estimate a proportion \(p\).

The formula is:

\[\hat{p} \pm z_{\alpha/2} \cdot \sqrt{\frac{\hat{p}(1-\hat{p})}{n}}\]

where:

  • \(\hat{p}\) is the sample proportion (number of successes / n)
  • \(z_{\alpha/2}\) is the standard normal critical value (not t, because the binomial is well-approximated by normal for large n)
  • \(\sqrt{\frac{\hat{p}(1-\hat{p})}{n}}\) is the standard error for a proportion

Important: When is the normal approximation valid?

The normal approximation works when both \(n\hat{p} \geq 5\) and \(n(1-\hat{p}) \geq 5\). This ensures we have enough successes and failures. If this condition is violated, use exact binomial methods or Wilson's score interval (more advanced).

Worked Example: Election Poll

A poll of 400 voters finds 204 support a candidate. What is the 95% CI for the true proportion?

import numpy as np from scipy import stats n = 400 successes = 204 p_hat = successes / n # Check conditions n_p = n * p_hat n_1_p = n * (1 - p_hat) print(f"Conditions: n*p_hat = {n_p:.1f}, n*(1-p_hat) = {n_1_p:.1f}") print(f"Normal approximation valid? {n_p >= 5 and n_1_p >= 5}") # Standard error se_prop = np.sqrt(p_hat * (1 - p_hat) / n) # Critical value (95% CI) z_crit = stats.norm.ppf(0.975) margin = z_crit * se_prop lower = p_hat - margin upper = p_hat + margin print(f"\np_hat = {p_hat:.4f}") print(f"SE = {se_prop:.4f}") print(f"95% CI: [{lower:.4f}, {upper:.4f}]") print(f"As percentages: [{lower*100:.2f}%, {upper*100:.2f}%]")
Conditions: n*p_hat = 204.0, n*(1-p_hat) = 196.0 Normal approximation valid? True p_hat = 0.5100 SE = 0.0249 95% CI: [0.4609, 0.5591] As percentages: [46.09%, 55.91%]

The poll shows 51% support (204 out of 400). The 95% CI is [46.09%, 55.91%]. Notice it includes 50% — we cannot rule out a true tie. The poll is consistent with anywhere from just under 46% to just under 56% support.

Standard Error Maximization: Why p̂ = 0.5 is Worst-Case

Notice something: the standard error depends on \(\hat{p}(1-\hat{p})\). This is maximized when \(\hat{p} = 0.5\). To see why, take the derivative:

\[\frac{d}{dp}\left[p(1-p)\right] = 1 - 2p = 0 \quad \Rightarrow \quad p = 0.5\]

This means:

  • Proportions near 0.5 have the largest standard error (widest CI)
  • Proportions near 0 or 1 have smaller standard errors (narrower CI)
  • For sample size planning, use p̂ = 0.5 as the worst case if you have no prior information

Binary outcomes are less informative near 50%

If you're surveying about support for a policy, a poll showing 51% support is noisier (wider CI) than a poll showing 70% support, even with the same sample size. This is because near 50%, people are most mixed — you need more data to distinguish true support from sampling noise. This is why prediction markets require larger samples when the outcome is near 50–50.

3.9 — Difference of Means

Often you want to compare two groups: does treatment group A have a higher mean than control group B?

The CI for the difference \((\bar{X}_1 - \bar{X}_2)\) is:

\[(\bar{X}_1 - \bar{X}_2) \pm t_{\text{df}} \cdot \sqrt{\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}}\]

The standard error combines the variability from both groups. If the CI includes zero, the two means are not significantly different at the chosen confidence level.

Important: The degrees of freedom calculation is more complex here. If the two groups have equal variance (an assumption we test separately), we use a pooled estimate. If they have unequal variances, we use Welch's correction. Python's scipy.stats.ttest_ind() and scipy.stats.t.interval() handle this automatically.

from scipy import stats import numpy as np # Simulate two groups: treatment vs. control np.random.seed(42) control = np.random.normal(loc=100, scale=15, size=50) treatment = np.random.normal(loc=108, scale=15, size=50) # Difference in means diff = treatment.mean() - control.mean() # 95% CI for the difference using ttest_ind t_stat, p_value = stats.ttest_ind(treatment, control) # For CI, compute standard error and use t-distribution se_diff = np.sqrt(control.var(ddof=1)/50 + treatment.var(ddof=1)/50) df = 98 # n1 + n2 - 2 for pooled case t_crit = stats.t.ppf(0.975, df=df) ci_lower = diff - t_crit * se_diff ci_upper = diff + t_crit * se_diff print(f"Control mean: {control.mean():.2f}") print(f"Treatment mean: {treatment.mean():.2f}") print(f"Difference: {diff:.2f}") print(f"95% CI: [{ci_lower:.2f}, {ci_upper:.2f}]") print(f"Includes zero? {ci_lower <= 0 <= ci_upper}")
Control mean: 100.23 Treatment mean: 107.31 Difference: 7.08 95% CI: [1.82, 12.34] Includes zero? False

The treatment increases the mean by about 7 points. The 95% CI [1.82, 12.34] does NOT include zero, so we can conclude at the 95% confidence level that there is a real difference.

When to trust the CI for difference of means

Both groups should be approximately normally distributed (or have large samples, n > 30), and observations within each group should be independent. If samples are very unequal in size or variance, use Welch's test (which doesn't assume equal variances). Always visualize the two distributions first — a summary statistic can hide important details.

Key Takeaways

Confidence intervals are ranges of plausible parameter values, not probabilities about a single interval. The standard error \(\frac{s}{\sqrt{n}}\) scales with \(\frac{1}{\sqrt{n}}\) — quadruple the sample to halve the margin of error. The t-distribution accounts for estimating the standard deviation from the sample. CI–test duality means a CI includes exactly those parameter values that would not be rejected in a test. Confidence and precision trade off: higher confidence requires a wider interval unless you collect more data. For proportions, uncertainty is highest near \(\hat{p} = 0.5\). Always visualize the data and check assumptions — summary statistics can hide important patterns.
