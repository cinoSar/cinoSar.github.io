10.1 — What Are Panel Data?

Panel data (also called longitudinal data) track multiple units over multiple time periods. Unlike cross-sectional data (many units, one period) or time-series data (one unit, many periods), panel data combine both dimensions: many units observed repeatedly.

Data Structures: A Comparison

Cross-sectional: One snapshot in time. Example: wages of workers in 2024.

Worker 1: Y₁, X₁ Worker 2: Y₂, X₂ Worker 3: Y₃, X₃ ...

Time-series: One unit over many periods. Example: GDP of France from 1980–2024.

Year 1980: Y₁₉₈₀ Year 1981: Y₁₉₈₁ ... Year 2024: Y₂₀₂₄

Panel: Many units, many periods. Example: firm revenue for 100 firms from 2015–2024.

Firm 1, 2015: Y₁,₂₀₁₅, X₁,₂₀₁₅ Firm 1, 2016: Y₁,₂₀₁₆, X₁,₂₀₁₆ ... Firm 2, 2015: Y₂,₂₀₁₅, X₂,₂₀₁₅ ... Firm 100, 2024: Y₁₀₀,₂₀₂₄, X₁₀₀,₂₀₂₄

Notation

We denote the outcome for unit \(i\) at time \(t\) as \(Y_{it}\), and the regressor as \(X_{it}\). With \(N\) units and \(T\) periods, we have \(N \times T\) total observations. For balanced panels, each unit appears in every time period; for unbalanced panels, some units drop out or enter later.

Panel Data in Practice

Firm performance over years: Track 500 firms' revenue, R&D spending, and profitability from 2010–2023 (N=500, T=14).

Country GDP across decades: Track 180 countries' GDP, trade openness, and governance quality from 1960–2024 (N=180, T=65).

Student test scores across semesters: Track 10,000 students' test scores and study hours across 8 semesters (N=10,000, T=8).

Creating Panel Data with Pandas MultiIndex

import pandas as pd import numpy as np # Create a simple panel dataset: 5 firms, 4 years n_firms = 5 n_years = 4 years = [2020, 2021, 2022, 2023] # Create MultiIndex index = pd.MultiIndex.from_product( [['Firm_A', 'Firm_B', 'Firm_C', 'Firm_D', 'Firm_E'], years], names=['firm_id', 'year'] ) # Generate random data np.random.seed(42) data = { 'revenue': np.random.uniform(100, 1000, 20), 'rd_spending': np.random.uniform(10, 100, 20), 'employees': np.random.uniform(100, 5000, 20) } df = pd.DataFrame(data, index=index) print(df) print("\nShape:", df.shape) print("N = ", n_firms, ", T = ", n_years)

Key Concept: Panel Data Are the Gold Standard in Applied Microeconometrics

Panel data are uniquely powerful because they let you control for unobserved heterogeneity without needing to find an instrument. By following the same units over time, you can eliminate time-invariant confounders (firm quality, ability, geography) and isolate causal effects with confidence. This is why panel data are so valuable in applied econometrics.

10.2 — The Problem: Unobserved Heterogeneity

Suppose we want to estimate the effect of R&D spending on firm productivity. A naive approach: run OLS on a cross-section of firms.

\[Y_i = \beta_0 + \beta_1 X_i + \varepsilon_i\]

where \(Y_i\) is firm productivity and \(X_i\) is R&D spending. But this creates a fundamental problem: better-managed firms both spend more on R&D and have higher productivity. Management quality is unobserved but correlated with both variables, causing omitted variable bias.

The Fixed Effects Model

With panel data, we can write:

\[Y_{it} = \beta_0 + \beta_1 X_{it} + \alpha_i + \varepsilon_{it}\]

The term \(\alpha_i\) is the individual effect (or entity effect): everything about unit \(i\) that is constant over time. This includes management quality, institutional culture, innate ability, geographic advantages—all the unobserved heterogeneity that doesn't change within the observation window.

Why OVB Still Occurs

If \(\text{Cov}(\alpha_i, X_{it}) \neq 0\)—that is, the time-invariant unobserved effect correlates with the regressor—then pooled OLS is biased. In our example, high-quality management (high \(\alpha_i\)) correlates with high R&D spending (\(X_{it}\)), inflating the estimated effect of R&D.

Simulation: Pooled OLS Is Biased

import numpy as np import pandas as pd import statsmodels.api as sm np.random.seed(123) # True model: Y_it = 2 + 0.5*X_it + alpha_i + epsilon_it # True beta_1 = 0.5 # alpha_i ~ N(0, 1), correlated with X_it n_firms = 50 n_years = 10 # Create panel structure firm_ids = np.repeat(np.arange(n_firms), n_years) year_ids = np.tile(np.arange(n_years), n_firms) # Unobserved heterogeneity (constant over time for each firm) alpha = np.random.normal(0, 1, n_firms) # X correlated with alpha: X_it = alpha_i + noise_it X = alpha[firm_ids] + np.random.normal(0, 0.5, n_firms * n_years) # Y: true effect of X is 0.5 Y = 2 + 0.5 * X + alpha[firm_ids] + np.random.normal(0, 0.3, n_firms * n_years) # Pooled OLS (biased) X_const = sm.add_constant(X) ols_model = sm.OLS(Y, X_const).fit() print("Pooled OLS (BIASED):") print("Coefficient on X:", ols_model.params[1].round(3)) print("True beta_1 = 0.5, but we estimated ~1.2 due to unobserved heterogeneity")

Result: The pooled OLS coefficient is ~1.2, far above the true value of 0.5. This is upward bias due to unobserved heterogeneity: high-quality firms spend more on R&D and are more productive, so the coefficient inflates.

10.3 — Fixed Effects (FE) Estimator

The Idea: Eliminate \(\alpha_i\) with Dummies

The simplest approach to deal with unobserved heterogeneity: add a dummy variable for each unit. The model becomes:

\[Y_{it} = \beta_0 + \beta_1 X_{it} + \sum_{i=1}^{N} \gamma_i D_i + \varepsilon_{it}\]

where \(D_i\) is a dummy variable equal to 1 for unit \(i\) and 0 otherwise. The dummies absorb all time-invariant differences between units, including \(\alpha_i\). Any time-invariant confounder is now "controlled for" by the dummies.

The Within Transformation (Demeaning)

An equivalent and computationally efficient approach: subtract unit means from all variables. This is called the within transformation.

\[\tilde{Y}_{it} = Y_{it} - \bar{Y}_i\]

\[\tilde{X}_{it} = X_{it} - \bar{X}_i\]

where \(\bar{Y}_i = \frac{1}{T} \sum_t Y_{it}\) and \(\bar{X}_i = \frac{1}{T} \sum_t X_{it}\) are unit means.

The model becomes:

\[\tilde{Y}_{it} = \beta_1 \tilde{X}_{it} + (\tilde{\varepsilon}_{it})\]

Notice that \(\alpha_i\) disappears! Since \(\alpha_i\) is constant over time, subtracting the unit mean eliminates it. This is the heart of fixed effects estimation: FE uses only within-unit variation—how variables change over time for the same unit—to identify \(\beta_1\).

Key Insight: Within-Unit Causal Logic

Fixed effects answer a different question than cross-sectional OLS:

Cross-sectional OLS: "Do firms that spend more on R&D tend to be more productive?" (Confounded by firm quality.)

Fixed Effects: "When a specific firm increases its R&D spending, what happens to its productivity?" (Causal: compares the same firm to itself over time.)

This is a much more credible causal estimate.

Important Limitation: Time-Invariant Variables

You cannot estimate the effect of any variable that does not change over time. For example, if you include gender (constant for each person) or industry sector (constant for each firm), these variables are absorbed by the fixed effects and drop out. If you need to estimate effects of time-invariant variables, you must use random effects (Section 10.4) or assume they are exogenous.

Estimating Fixed Effects with linearmodels

from linearmodels.panel import PanelOLS import pandas as pd # Assuming df is a panel dataset with MultiIndex (firm_id, year) # and columns: revenue, rd_spending, employees # Set up the model with entity fixed effects model_fe = PanelOLS( df['revenue'], df[['rd_spending', 'employees']], entity_effects=True # Entity (firm) fixed effects ) results_fe = model_fe.fit(cov_type='robust') print(results_fe) # Compare with pooled OLS (no fixed effects) import statsmodels.api as sm X = sm.add_constant(df[['rd_spending', 'employees']]) model_ols = sm.OLS(df['revenue'], X) results_ols = model_ols.fit() print("\nCoefficient on rd_spending:") print(" OLS (pooled):", results_ols.params[1].round(3)) print(" FE (within):", results_fe.params[0].round(3))

Key Concept: Fixed Effects Isolate Within-Unit Variation

The fixed effects estimator answers: "When a specific firm increases its R&D spending by 1 unit, how much does its productivity increase?" rather than "Do firms that spend more on R&D tend to be more productive?" The distinction is crucial. The first question has causal interpretation; the second is confounded by unobserved firm quality. Fixed effects eliminate time-invariant confounders and allow credible causal inference.

10.4 — Random Effects (RE) Estimator

The RE Model

Random effects offers an alternative assumption: treat \(\alpha_i\) as a random variable that is uncorrelated with the regressors.

\[\text{Cov}(\alpha_i, X_{it}) = 0\]

This is a strong assumption, but if it holds, RE is more efficient (has smaller standard errors) than FE because it uses both within-unit and between-unit variation.

The RE Estimator

RE uses a Generalized Least Squares (GLS) approach: a weighted combination of the between estimator (using unit means) and the within estimator (using deviations from unit means). The weights depend on the estimated variance components.

The key advantage of RE over FE: you can estimate effects of time-invariant variables (gender, birthplace, sector) because the dummies are not included.

When Is the RE Assumption Plausible?

In experimental or quasi-experimental settings, where treatment assignment is random and independent of unit characteristics. In observational data with economic units (firms, people, countries), the assumption is rarely credible: better-managed firms or more-capable workers will have both higher unobserved quality and higher values of most regressors.

Estimating RE

from linearmodels.panel import RandomEffects # Estimate the same model with random effects model_re = RandomEffects( df['revenue'], df[['rd_spending', 'employees']] ) results_re = model_re.fit(cov_type='robust') print("Random Effects Results:") print(results_re) # Compare FE and RE coefficients print("\nCoefficient Comparison:") print(" FE: ", results_fe.params[0].round(3)) print(" RE: ", results_re.params[0].round(3))

In this example, if FE and RE coefficients differ substantially, it suggests that \(\text{Cov}(\alpha_i, X_{it}) \neq 0\), and RE is biased.

10.5 — Fixed vs Random Effects: The Hausman Test

Motivation

When should you use FE and when RE? The Hausman test provides a formal answer.

The Test

Null hypothesis: \(H_0: \text{Cov}(\alpha_i, X_{it}) = 0\) (RE is consistent)

Alternative hypothesis: \(H_1: \text{Cov}(\alpha_i, X_{it}) \neq 0\) (RE is inconsistent, use FE)

Test statistic:

\[H = (\hat{\beta}_{\text{FE}} - \hat{\beta}_{\text{RE}})' [\text{Var}(\hat{\beta}_{\text{FE}}) - \text{Var}(\hat{\beta}_{\text{RE}})]^{-1} (\hat{\beta}_{\text{FE}} - \hat{\beta}_{\text{RE}}) \sim \chi^2(k)\]

If the p-value is small (reject \(H_0\)), use FE. If the p-value is large (fail to reject \(H_0\)), RE is more efficient.

Practical Guidance

In applied econometrics, FE is the default. The assumption that unobserved heterogeneity is uncorrelated with regressors is rarely credible in observational data. Economists use FE unless they have strong theoretical reasons to believe in the RE assumption (e.g., experimental settings).

Conducting the Hausman Test

from scipy.stats import chi2 # Hausman test statistic (simplified) # In practice, use dedicated functions in linearmodels or statsmodels beta_fe = results_fe.params beta_re = results_re.params var_fe = results_fe.cov var_re = results_re.cov # Compute the test statistic diff = beta_fe - beta_re var_diff = var_fe - var_re try: H = diff.T @ np.linalg.inv(var_diff) @ diff df = len(beta_fe) p_value = 1 - chi2.cdf(H, df) print(f"Hausman Test:") print(f" Test Statistic: {H:.3f}") print(f" p-value: {p_value:.3f}") if p_value < 0.05: print(" Reject H0: Use FE (endogeneity likely)") else: print(" Fail to reject H0: RE is preferred (more efficient)") except: print("Matrix inversion failed; variance-covariance may be singular.")

Key Concept: In Applied Economics, FE Is the Safe Default

The assumption underlying random effects—that individual effects are uncorrelated with regressors—is rarely credible in observational data with economic units. Use FE as your default estimator unless you have strong theoretical or empirical reasons to believe the RE assumption holds (e.g., in experimental or quasi-experimental settings).

10.6 — Time Fixed Effects

The Problem Time Fixed Effects Solve

So far, we have discussed entity (unit) fixed effects, which control for time-invariant unit characteristics. But sometimes there are shocks or changes common to all units at a given time—economic recessions, policy changes, technological innovations, business cycles.

To control for these, we add time fixed effects:

\[Y_{it} = \beta_1 X_{it} + \alpha_i + \lambda_t + \varepsilon_{it}\]

where \(\lambda_t\) is the time effect: the part of the outcome common to all units in year \(t\).

Two-Way Fixed Effects

Including both entity and time effects is called two-way fixed effects (or "two-way FE"). This model controls for both time-invariant unit heterogeneity and unit-invariant time effects.

Example: Firm Policy and Macroeconomic Conditions

Suppose you estimate the effect of a firm-level policy (e.g., remote work adoption) on productivity. Entity FE controls for firm quality; time FE controls for macroeconomic conditions (GDP growth, industry cycles) that affect all firms.

Estimating Two-Way FE

from linearmodels.panel import PanelOLS # Estimate model with both entity and time fixed effects model_twoway = PanelOLS( df['revenue'], df[['rd_spending', 'employees']], entity_effects=True, # Firm fixed effects (alpha_i) time_effects=True # Year fixed effects (lambda_t) ) results_twoway = model_twoway.fit(cov_type='robust') print("Two-Way Fixed Effects Results:") print(results_twoway) # Compare coefficients across specifications print("\nCoefficient on rd_spending across models:") print(" Pooled OLS: ", results_ols.params[1].round(3)) print(" Entity FE: ", results_fe.params[0].round(3)) print(" Two-Way FE: ", results_twoway.params[0].round(3))

Often, the time effects are not of direct interest but are included to control for confounding. You can examine the time effects (coefficients on the time dummies) if you want to understand macro trends.

10.7 — Clustered Standard Errors

The Problem: Serial Correlation in Panel Data

In panel data, errors are typically correlated within units over time. If a firm experiences a positive productivity shock in one year, that shock likely persists into the next year. Formally:

\[\text{Corr}(\varepsilon_{it}, \varepsilon_{is}) \neq 0 \quad \text{for} \quad t \neq s\]

This is called serial correlation or autocorrelation within clusters (units).

Consequence: Standard Errors Are Biased

OLS (and FE, RE) standard errors assume errors are uncorrelated. If this assumption is violated, reported standard errors are too small. You over-reject null hypotheses and report spuriously significant coefficients. This is a critical problem in practice.

Solution: Cluster at the Entity Level

Always cluster standard errors at the entity (unit) level in panel data. This allows arbitrary correlation within units over time while assuming independence across units.

Implementing Clustered Standard Errors

from linearmodels.panel import PanelOLS # Estimate FE with clustered standard errors model_fe = PanelOLS( df['revenue'], df[['rd_spending', 'employees']], entity_effects=True ) # Fit with clustered standard errors at entity level results_fe_clustered = model_fe.fit( cov_type='clustered', cluster_entity=True ) print(results_fe_clustered) # Compare standard errors with non-clustered results_fe_robust = model_fe.fit(cov_type='robust') print("\nComparison of Standard Errors:") print(" Robust Clustered") for i in range(len(results_fe_robust.params)): print(f"Var {i}: {results_fe_robust.std_errors[i]:.4f} {results_fe_clustered.std_errors[i]:.4f}")

Notice that clustered standard errors are typically larger than robust standard errors, reflecting the within-unit serial correlation. Ignoring this would lead to over-confident inference.

Key Concept: Always Cluster Standard Errors in Panel Data

In panel data, ALWAYS cluster standard errors at the entity level. Serial correlation within units is the norm, not the exception. Failure to cluster leads to biased standard errors, incorrect confidence intervals, and false significance. This is non-negotiable in modern applied econometrics.

10.8 — Practical Workflow for Panel Data

Step-by-Step Approach

1. Explore the data: How many units? How many periods? Balanced or unbalanced?

print("Data Overview:") print(f" Shape: {df.shape}") print(f" Units (N): {df.index.get_level_values('firm_id').nunique()}") print(f" Periods (T): {df.index.get_level_values('year').nunique()}") print(f" Balanced: {df.index.get_level_values('firm_id').nunique() * df.index.get_level_values('year').nunique() == len(df)}")

2. Baseline: Pooled OLS. Run a naive regression to understand direction and magnitude.

3. Run FE model with entity effects. Compare coefficients with OLS. Are they substantially different?

4. Run RE model. Compare RE and FE coefficients.

5. Hausman test. FE vs RE: which is appropriate?

6. Add time effects if appropriate. Do macro shocks or time trends confound the relationship?

7. Always cluster standard errors. At the entity level, unless theoretical considerations suggest clustering at a different level (e.g., state or industry).

Connection to Module 09: Endogeneity

Fixed effects solves endogeneity due to time-invariant confounders. But if endogeneity remains—if there is time-varying unobserved heterogeneity or reverse causality—you need panel IV: combine FE with instrumental variables. This extends the toolkit from Module 09 to the panel setting.

Connection to Module 11: Difference-in-Differences

Difference-in-Differences (DiD) is a special case of two-way fixed effects. It exploits a policy change (treatment) that affects some units but not others, using the parallel trends assumption to identify causal effects. The two-way FE model you learned here provides the foundation for DiD estimation.

Key Takeaways

  • Panel data combine cross-sectional and time-series variation. They are uniquely powerful for causal inference because they allow control for unobserved heterogeneity without instruments.
  • Fixed effects control for time-invariant unit heterogeneity by using the within transformation (demeaning). FE estimates are based on variation within units over time, providing causal identification.
  • FE cannot estimate effects of time-invariant variables because they are absorbed by the fixed effects. Time-invariant confounders are controlled, but time-invariant regressors cannot have estimated effects.
  • Random effects assumes unobserved heterogeneity is uncorrelated with regressors. This is a strong assumption, rarely credible in observational economic data. Use RE only when theory strongly supports it (e.g., experimental settings).
  • The Hausman test compares FE and RE. Reject H₀ (use FE) if the test is significant, suggesting endogeneity. In practice, economists default to FE.
  • Time fixed effects control for unit-invariant shocks. Two-way FE controls for both time-invariant unit effects and unit-invariant time effects, absorbing macroeconomic confounders.
  • Always cluster standard errors at the entity level in panel data. Serial correlation within units is the norm. Failure to cluster biases standard errors downward and leads to false significance.
  • If time-varying endogeneity remains, use panel IV (extend Module 09 to panels). If a policy causes treatment assignment, use difference-in-differences (Module 11).
