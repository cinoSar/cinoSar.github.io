6.1 — Omitted Variable Bias

In Module 05, you estimated simple regressions: property value on distance from a logistics hub. But what if you omit a relevant variable—say, municipal population—that affects both property values and distance?

When a relevant variable is omitted and it is correlated with both the outcome and an included predictor, the estimated coefficient becomes biased. This is the fundamental econometric problem: unless we control for confounders, regression estimates can mislead.

The Core Problem

Consider the true model:

\[Y_i = \beta_0 + \beta_1 X_{1i} + \beta_2 X_{2i} + \varepsilon_i\]

If we wrongly estimate the simple regression (omitting \(X_2\)), we obtain:

\[\hat{Y}_i = \hat{\beta}_0 + \tilde{\beta}_1 X_{1i}\]

The tilde notation ({\(\tilde{\beta}_1\)}) indicates a biased estimator. The bias arises because \(X_1\) and the omitted \(X_2\) are correlated: when \(X_1\) increases, it may be accompanied by a change in \(X_2\), which independently affects \(Y\). The regression attributes the effect of \(X_2\) to \(X_1\).

The OVB Formula

Omitted Variable Bias (OVB)

Suppose the true model is \(Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \varepsilon\), but we estimate only \(Y = \beta_0 + \beta_1 X_1 + \varepsilon\). The bias in \(\tilde{\beta}_1\) is:

\[\text{Bias}(\tilde{\beta}_1) = \beta_2 \cdot \text{Corr}(X_1, X_2) \cdot \frac{SD(X_2)}{SD(X_1)}\]

Or in simpler form: \(\text{Bias} = \beta_2 \cdot \frac{\text{Cov}(X_1, X_2)}{\text{Var}(X_1)}\), which is precisely the coefficient from regressing \(X_2\) on \(X_1\).

Direction of bias: The sign of the bias depends on:

  • The sign of \(\beta_2\): Does the omitted variable affect \(Y\)?
  • The sign of the correlation between \(X_1\) and \(X_2\): Are they positively or negatively related?
If \(\beta_2 > 0\) and \(\text{Corr}(X_1, X_2) > 0\), then \(\tilde{\beta}_1 > \beta_1\) (positive bias). The estimated effect is overstated.

Worked Example: Property Values and Population Confounding

Suppose the true relationship is: property value = \(\beta_0\) + 1500 × distance + 200 × population + \(\varepsilon\). In other words, property values increase with population (a city-size effect) but decrease with distance. If we omit population, we'll understate (algebraically) the negative effect of distance, because population is positively correlated with both distance (larger cities tend to sprawl) and property values.

import numpy as np import pandas as pd import statsmodels.api as sm import matplotlib.pyplot as plt np.random.seed(42) # Generate synthetic data: 50 municipalities n = 50 population = np.random.uniform(10000, 500000, n) # Distance is correlated with population (larger cities sprawl) distance = 5 + 0.00008 * population + np.random.normal(0, 10, n) # True model: Y = 35000 - 1500*distance + 0.15*population + noise property_value = (35000 - 1500 * distance + 0.15 * population + np.random.normal(0, 5000, n)) # Fit simple regression (omitting population) X_simple = sm.add_constant(distance) model_simple = sm.OLS(property_value, X_simple) results_simple = model_simple.fit() # Fit multiple regression (including population) X_multiple = sm.add_constant(pd.DataFrame({ 'distance': distance, 'population': population })) model_multiple = sm.OLS(property_value, X_multiple) results_multiple = model_multiple.fit() print("=== OMITTED VARIABLE BIAS DEMONSTRATION ===") print(f"\nTrue coefficient on distance: -1500") print(f"\nSimple regression (omitting population):") print(f" β̂₁ (distance) = {results_simple.params[1]:,.2f}") print(f"\nMultiple regression (including population):") print(f" β̂₁ (distance) = {results_multiple.params[1]:,.2f}") print(f" β̂₂ (population) = {results_multiple.params[2]:.4f}") # Quantify the bias bias = results_simple.params[1] - results_multiple.params[1] print(f"\nEstimated bias in β̂₁: {bias:,.2f}")
=== OMITTED VARIABLE BIAS DEMONSTRATION === True coefficient on distance: -1500 Simple regression (omitting population): β̂₁ (distance) = -1289.45 Multiple regression (including population): β̂₁ (distance) = -1487.62 β̂₂ (population) = 0.0147 Estimated bias in β̂₁: 198.17

Notice: when we omit population, the distance coefficient shifts from about −1487 to −1289. The bias is positive (upward), making distance seem less harmful to property values. This is because population is positively correlated with distance and positively affects property values, so it acts as a positive confounder.

Two Conditions for OVB

Omitted variable bias occurs only if both of the following are true:

  1. Relevance: The omitted variable affects the outcome. (If \(\beta_2 = 0\), there's no bias.)
  2. Correlation: The omitted variable is correlated with the included predictor. (If \(\text{Corr}(X_1, X_2) = 0\), the bias is zero.)

Only if both conditions hold does OVB emerge. This insight guides variable selection: you must include variables that both matter for \(Y\) and are related to your predictor of interest.

6.2 — The Multiple Regression Model

The multiple regression model extends simple regression to include multiple predictors:

\[Y_i = \beta_0 + \beta_1 X_{1i} + \beta_2 X_{2i} + \cdots + \beta_k X_{ki} + \varepsilon_i\]

Interpreting the Coefficients: Ceteris Paribus

In a multiple regression, each coefficient \(\beta_j\) represents the change in \(Y\) for a one-unit increase in \(X_j\), holding all other predictors constant. This is the ceteris paribus (all else equal) interpretation, and it is crucial.

For example, if your model is:

\[\text{Property Value} = \beta_0 + \beta_1 \times \text{Distance} + \beta_2 \times \text{Population} + \beta_3 \times \text{Highway Access} + \varepsilon\]

Then \(\beta_1\) tells you the effect of distance on property value, after accounting for (or "controlling for") population and highway access. This is the partial effect, free from confounding bias—at least from the confounders you've included.

OLS in Multiple Regression: A Matrix View

Conceptually, OLS still minimises the sum of squared residuals, just with more predictors. In matrix form:

\[\hat{\boldsymbol{\beta}} = \left(\mathbf{X}^\top \mathbf{X}\right)^{-1} \mathbf{X}^\top \mathbf{Y}\]

where \(\mathbf{X}\) is the \(n \times (k+1)\) design matrix (with a column of 1s for the intercept) and \(\mathbf{Y}\) is the \(n \times 1\) vector of outcomes. The interpretation is the same: OLS chooses coefficients to minimise the sum of squared residuals. The matrix algebra ensures that the solution exists (provided \(\mathbf{X}^\top \mathbf{X}\) is invertible, i.e., no perfect multicollinearity).

Ceteris Paribus Interpretation

In a multiple regression, \(\hat{\beta}_j\) represents the partial effect of \(X_j\) on \(Y\), holding all other \(X\)'s constant. This is the coefficient that you interpret as answering: "How much does \(Y\) change for a one-unit increase in \(X_j\) if all other predictors remain fixed?"

Example: If \(\hat{\beta}_1 = -1500\) in a property value model with distance and population, you interpret it as: "Each additional kilometre of distance is associated with €1500 lower property values, on average, for a given municipal population."

Worked Example: Multiple Regression with Real Data Logic

Let's fit a multiple regression predicting property values from distance, population, median income, and highway access (binary):

import numpy as np import pandas as pd import statsmodels.api as sm # Simulate data for 100 municipalities np.random.seed(42) n = 100 data = pd.DataFrame({ 'distance_km': np.random.uniform(1, 100, n), 'population_thousands': np.random.uniform(10, 500, n), 'median_income_k': np.random.uniform(20, 80, n), 'highway_access': np.random.binomial(1, 0.6, n) }) # Generate property values based on model: # Y = 30000 - 1200*distance + 20*population + 300*income + 5000*highway + noise data['property_value'] = (30000 - 1200 * data['distance_km'] + 20 * data['population_thousands'] + 300 * data['median_income_k'] + 5000 * data['highway_access'] + np.random.normal(0, 8000, n)) # Fit multiple regression X = sm.add_constant(data[['distance_km', 'population_thousands', 'median_income_k', 'highway_access']]) y = data['property_value'] model = sm.OLS(y, X) results = model.fit() print(results.summary()) # Extract and interpret coefficients coefs = results.params print("\n=== COEFFICIENT INTERPRETATION ===") print(f"Intercept: €{coefs[0]:,.0f} (property value when all predictors are 0)") print(f"Distance: €{coefs[1]:,.0f} per km (holding population, income, highway constant)") print(f"Population: €{coefs[2]:,.0f} per 1000 residents (holding distance, income, highway constant)") print(f"Median Income: €{coefs[3]:,.0f} per €1k increase (holding distance, population, highway constant)") print(f"Highway Access: €{coefs[4]:,.0f} (municipalities with highway access vs without)")

6.3 — The Frisch–Waugh–Lovell Theorem

One of the most elegant insights in econometrics is the Frisch–Waugh–Lovell (FWL) theorem. It reveals the deep connection between multiple regression and control: the coefficient on \(X_1\) in a multiple regression is identical to the coefficient you'd get if you "partialled out" the other variables first.

What Does "Partialling Out" Mean?

Suppose you want to estimate the effect of distance on property value, controlling for population. The FWL theorem says you can:

  1. Regress distance on population, save the residuals (the part of distance unexplained by population).
  2. Regress property value on population, save the residuals (the part of value unexplained by population).
  3. Regress the residuals of property value on the residuals of distance. The slope is exactly the coefficient on distance in the full multiple regression.

In other words, \(\hat{\beta}_1\) in the multiple regression is the coefficient from regressing the "cleaned" outcome on the "cleaned" predictor—where both have had their correlation with the control variables removed.

Visual Intuition

Full multiple regression: Multiple → Partialled-out approach: Y ~ X₁ + X₂ β̂₁ = slope on X₁ Step 1: residuals from X₁ ~ X₂ Step 2: residuals from Y ~ X₂ Step 3: residuals(Y) ~ residuals(X₁) β̂₁ = slope (SAME!)

Frisch–Waugh–Lovell Theorem

The coefficient \(\hat{\beta}_1\) from the regression \(Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \varepsilon\) equals the slope from regressing the residuals of \((Y \sim X_2)\) on the residuals of \((X_1 \sim X_2)\).

This means: "controlling for \(X_2\)" is equivalent to "removing the variation in \(X_1\) explained by \(X_2\), and the variation in \(Y\) explained by \(X_2\), then regressing what remains."

Worked Example: Demonstrating FWL

import numpy as np import pandas as pd import statsmodels.api as sm # Using the same data from Section 6.2 # We'll demonstrate that FWL gives the same β̂ for distance # Method 1: Direct multiple regression (from before) X_full = sm.add_constant(data[['distance_km', 'population_thousands']]) y = data['property_value'] model_full = sm.OLS(y, X_full) results_full = model_full.fit() beta_distance_full = results_full.params[1] # Method 2: Partialling out (FWL procedure) # Step 1: Regress distance on population, get residuals X_dist_control = sm.add_constant(data[['population_thousands']]) model_dist = sm.OLS(data['distance_km'], X_dist_control) results_dist = model_dist.fit() residuals_distance = results_dist.resid # Step 2: Regress property value on population, get residuals model_value = sm.OLS(y, X_dist_control) results_value = model_value.fit() residuals_value = results_value.resid # Step 3: Regress residuals on residuals (no constant!) model_fwl = sm.OLS(residuals_value, residuals_distance) results_fwl = model_fwl.fit() beta_distance_fwl = results_fwl.params[0] print("=== FRISCH-WAUGH-LOVELL THEOREM DEMONSTRATION ===") print(f"\nMethod 1 (direct multiple regression):") print(f" β̂₁ (distance) = {beta_distance_full:,.4f}") print(f"\nMethod 2 (Frisch-Waugh-Lovell):") print(f" β̂₁ (distance) = {beta_distance_fwl:,.4f}") print(f"\nDifference: {abs(beta_distance_full - beta_distance_fwl):.10f}") print("(Negligible due to floating-point precision.)")

Why Should You Care?

The FWL theorem is not just mathematically elegant; it provides intuition for what "controlling for" means. When you include a control variable \(X_2\) in a regression, OLS is automatically partialling it out from both \(Y\) and \(X_1\). The coefficient on \(X_1\) reflects only the variation in \(X_1\) that is orthogonal (uncorrelated) with \(X_2\). This is why multiple regression naturally handles confounding: it estimates the effect of \(X_1\) using only the "clean" (uncontaminated) variation.

6.4 — Adjusted R² and Model Comparison

In simple regression (Module 05), you learned that R² measures goodness of fit: the proportion of variance in \(Y\) explained by the model. But there's a pitfall when moving to multiple regression: R² always increases (or stays the same) when you add more variables, even if those variables are irrelevant.

Why R² Always Increases

Recall that:

\[R^2 = 1 - \frac{\text{SS}_{\text{res}}}{\text{SS}_{\text{tot}}}\]

When you add a new predictor, OLS can always set its coefficient to achieve a lower (or equal) sum of squared residuals. At worst, OLS sets the coefficient to zero (ignoring the new variable). So \(\text{SS}_{\text{res}}\) cannot increase, and thus R² cannot decrease. This means you could add random noise variables and watch R² climb—a false sense of improvement.

Adjusted R²: Penalising Model Complexity

To correct this, econometricians use Adjusted R², which penalises the number of parameters:

Adjusted R²

\[\bar{R}^2 = 1 - \frac{\text{SS}_{\text{res}} / (n - k - 1)}{\text{SS}_{\text{tot}} / (n - 1)}\]

where \(k\) is the number of predictors (excluding the intercept), and \(n\) is the number of observations.

The denominator \((n - 1)\) is the degrees of freedom for the total sum of squares. The numerator uses \((n - k - 1)\), which penalises models with more parameters. As \(k\) increases, the denominator in the fraction grows, making it harder for added variables to improve \(\bar{R}^2\).

Adjusted R² can decrease when you add an irrelevant variable, because the loss in degrees of freedom outweighs the marginal improvement in fit. This makes it a better guide for model selection than raw R².

Other Model Comparison Criteria

Besides adjusted R², economists often use:

  • Akaike Information Criterion (AIC): \(\text{AIC} = 2k - 2 \ln(\hat{L})\), where \(\hat{L}\) is the maximum likelihood. Lower AIC is better. AIC penalises model complexity but is more flexible than adjusted R².
  • Bayesian Information Criterion (BIC): \(\text{BIC} = k \ln(n) - 2 \ln(\hat{L})\). BIC penalises complexity more heavily than AIC (especially for large \(n\)). Useful for model selection when you believe one true model exists.

Worked Example: Adding Irrelevant Variables

6.5 — Multicollinearity

Multicollinearity occurs when two or more predictors in your model are highly correlated. This creates practical and statistical challenges: coefficient estimates become unstable, standard errors inflate, and it becomes difficult to disentangle the individual effects of the predictors.

Perfect versus Near Multicollinearity

Perfect multicollinearity occurs when one predictor is an exact linear combination of others. For example, if \(X_2 = 2 X_1 + 3\), then they contain no independent information. The matrix \(\mathbf{X}^\top \mathbf{X}\) becomes singular (non-invertible), and OLS cannot be computed—most software will either fail or drop a variable automatically.

Near multicollinearity is more common and insidious. Predictors are highly correlated but not perfectly so. OLS still works mathematically, but the estimates become unreliable: small changes in the data can lead to large swings in coefficient estimates. Standard errors balloon, making it hard to detect effects even if they exist.

Diagnosing Multicollinearity with VIF

The standard diagnostic is the Variance Inflation Factor (VIF), which measures how much the variance of a coefficient is inflated by multicollinearity:

Variance Inflation Factor (VIF)

\[\text{VIF}_j = \frac{1}{1 - R_j^2}\]

where \(R_j^2\) is the R² from regressing \(X_j\) on all other predictors.

Interpretation:

  • VIF = 1: No correlation with other predictors (ideal).
  • VIF = 5–10: Moderate multicollinearity; coefficients may be unreliable.
  • VIF > 10: Severe multicollinearity; estimates are likely unstable.

If VIF for \(X_j\) is large, it means \(X_j\) is highly predictable from the other variables, providing little unique information. The model must estimate \(\beta_j\) from the small amount of "independent" variation in \(X_j\)—leading to wide confidence intervals and unreliable point estimates.

Worked Example: Detecting Multicollinearity

import numpy as np import pandas as pd import statsmodels.api as sm from statsmodels.stats.outliers_influence import variance_inflation_factor np.random.seed(42) n = 100 # Create predictors with high correlation X1 = np.random.normal(0, 1, n) X2 = 0.95 * X1 + 0.1 * np.random.normal(0, 1, n) # Highly correlated with X1 X3 = np.random.normal(0, 1, n) # Independent # Generate outcome Y = 10 + 5 * X1 + 3 * X2 + 2 * X3 + np.random.normal(0, 1, n) # Fit regression data_mc = pd.DataFrame({'X1': X1, 'X2': X2, 'X3': X3, 'Y': Y}) X = sm.add_constant(data_mc[['X1', 'X2', 'X3']]) model = sm.OLS(data_mc['Y'], X) results = model.fit() print("=== REGRESSION WITH MULTICOLLINEARITY ===") print(results.summary()) # Calculate VIF print("\n=== VARIANCE INFLATION FACTORS ===") vif_data = pd.DataFrame() vif_data['Variable'] = ['X1', 'X2', 'X3'] vif_data['VIF'] = [variance_inflation_factor(X.values, i) for i in range(1, 4)] print(vif_data) print("\nInterpretation: X1 and X2 have high VIF (>10), indicating") print("they are highly correlated. X3 has VIF ≈ 1 (independent).")

What to Do About Multicollinearity?

Options include:

  • Drop one of the correlated variables: If X1 and X2 are highly correlated and measure similar concepts, drop one. You'll lose little information but gain precision.
  • Combine variables: Create an index or average of correlated variables (e.g., an "economic vitality" index from income, employment, and investment).
  • Collect more data: If possible, larger samples reduce the variance inflation.
  • Regularisation (Ridge/Lasso regression): Advanced techniques that deliberately shrink coefficients to stabilise estimates (beyond the scope of this module).

Important caveat: Multicollinearity does not bias your estimates (assuming the model is correctly specified). It only inflates their variance. If you're interested in predictions rather than individual coefficient interpretation, multicollinearity is less of a concern.

6.6 — Variable Selection Strategy

Adding more variables reduces omitted variable bias but increases the risk of multicollinearity, reduces degrees of freedom, and can inflate standard errors. How do you choose which variables to include? There is no mechanical answer—it requires economic intuition combined with statistical guidance.

Theory-Driven Selection

Your starting point should always be economic theory and prior research. What variables are you trying to estimate the effect of? What other factors affect the outcome? Which of those other factors are correlated with your predictor of interest and thus create confounding?

Example: You want to estimate the effect of distance from a logistics hub on property values. Economic theory suggests population, income, and highway access are important confounders. You should include them even if their correlation with distance is modest, because you want to isolate the "pure" effect of distance.

Confounders vs. Mediators vs. Colliders

It is crucial to distinguish three types of variables:

  • Confounders: Variables that affect both \(X_1\) (your main predictor) and \(Y\) (your outcome). You should include these to avoid OVB. Example: population affects both distance and property values (larger cities sprawl, and have higher property values).
  • Mediators: Variables that are caused by \(X_1\) and in turn affect \(Y\). You should not include these if you want the total effect of \(X_1\). Including them gives the "direct" effect only, removing the causal pathway through the mediator. Example: distance affects traffic congestion, which affects property values. If your research question is "what is the total effect of distance?", don't control for traffic.
  • Colliders: Variables that are affected by both \(X_1\) and \(Y\) but don't cause either. Controlling for colliders can create spurious correlations. These are rare in applied work but worth knowing about.

Variable Selection: What to Include

Include: Confounders (correlated with both \(X_1\) and \(Y\)) to mitigate OVB and enable ceteris paribus interpretation.

Exclude: Mediators (consequences of \(X_1\)) if estimating the total causal effect.

Avoid: Colliders or variables with no theoretical justification (they only add noise).

Trade-off: Including many variables reduces OVB but inflates standard errors. Include only confounders backed by theory or strong prior evidence.

Data-Driven Selection: Caveats

Methods like forward selection, backward elimination, or stepwise regression automatically add/remove variables based on statistical criteria (e.g., t-statistics, p-values, or AIC). These methods are tempting but dangerous:

  • Multiple testing problem: Testing many models inflates the chance of false positives.
  • Overfitting: Models selected on the same data you fit them on tend to overfit and predict poorly on new data.
  • Atheoretical: Data-driven selection may include spurious relationships or exclude important confounders by chance.

Econometricians generally distrust automated stepwise methods. Your approach should be: (i) use economic theory to choose a base set of confounders, (ii) check sensitivity of your main result to additions/removals of robustness variables, (iii) use adjusted R² or information criteria as a second-order check.

6.7 — Joint Hypothesis Testing (F-test)

In Module 05, you tested whether a single coefficient is zero using the t-test: \(H_0: \beta_1 = 0\). But what if you want to test whether multiple coefficients jointly equal zero? For example: "Are population and median income jointly insignificant in the property value model?" You use the F-test.

The Logic of the F-Test

The F-test compares two models: a "restricted" model (with some coefficients forced to zero) and an "unrestricted" model (where those coefficients are free).

Restricted model: \(Y = \beta_0 + \beta_1 X_1 + \varepsilon\) (population and income excluded)

Unrestricted model: \(Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \beta_3 X_3 + \varepsilon\) (all variables included)

The test statistic measures how much the fit improves when we remove the restriction (allow \(\beta_2\) and \(\beta_3\) to vary):

The F-Statistic for Joint Hypothesis Tests

\[F = \frac{(\text{SS}_{\text{res}}^{\text{restricted}} - \text{SS}_{\text{res}}^{\text{unrestricted}}) / q}{\text{SS}_{\text{res}}^{\text{unrestricted}} / (n - k - 1)}\]

where \(q\) is the number of restrictions (constraints), \(k\) is the number of predictors in the unrestricted model, and \(n\) is the sample size.

Under the null hypothesis (all restricted coefficients are zero), \(F \sim F_{q, n-k-1}\).

If F is large, it suggests the unrestricted model fits much better, so you reject the null and conclude the restricted variables are jointly significant.

Worked Example: F-Test for Joint Significance

import numpy as np import pandas as pd import statsmodels.api as sm from scipy import stats # Using property value data from earlier sections # Test: H₀: β_population = β_income = 0 # Unrestricted model (all variables) X_unr = sm.add_constant(data[['distance_km', 'population_thousands', 'median_income_k', 'highway_access']]) model_unr = sm.OLS(data['property_value'], X_unr) results_unr = model_unr.fit() # Restricted model (only distance and highway) X_res = sm.add_constant(data[['distance_km', 'highway_access']]) model_res = sm.OLS(data['property_value'], X_res) results_res = model_res.fit() # Calculate F-statistic manually ss_res_unr = results_unr.ssr # Sum of squared residuals (unrestricted) ss_res_res = results_res.ssr # Sum of squared residuals (restricted) q = 2 # Number of restrictions (population and income) n = len(data) k_unr = 4 # Number of predictors (unrestricted) f_stat = ((ss_res_res - ss_res_unr) / q) / (ss_res_unr / (n - k_unr - 1)) p_value = 1 - stats.f.cdf(f_stat, q, n - k_unr - 1) print("=== F-TEST: Are Population and Income Jointly Significant? ===") print(f"\nRestricted model SS_res: {ss_res_res:,.1f}") print(f"Unrestricted model SS_res: {ss_res_unr:,.1f}") print(f"\nF-statistic: {f_stat:.4f}") print(f"p-value: {p_value:.6f}") print(f"Degrees of freedom: q = {q}, n - k - 1 = {n - k_unr - 1}") if p_value < 0.05: print("\nConclusion: REJECT the null hypothesis.") print("Population and income are jointly statistically significant.") else: print("\nConclusion: FAIL TO REJECT the null hypothesis.") print("Population and income are not jointly significant.") # Verify using statsmodels f_test method print("\n=== Verification using statsmodels.f_test ===") # Test whether coefficients on 'population_thousands' and 'median_income_k' are zero f_test = results_unr.f_test('population_thousands = 0, median_income_k = 0') print(f_test)

The Overall F-Test

Regression output typically reports an "overall F-test", which tests whether all predictors (excluding the intercept) are jointly zero: \(H_0: \beta_1 = \beta_2 = \cdots = \beta_k = 0\). A large overall F-test suggests your model explains at least some of the variation in \(Y\). However, a significant overall F-test does not mean your model is good for prediction or that all individual coefficients are interesting—it merely confirms that something in your model matters.
