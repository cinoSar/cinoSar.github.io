8.1 — Why Linear Regression Fails for Binary Outcomes

Ordinary Least Squares (OLS) regression is a powerful tool for continuous outcomes. But when your dependent variable is binary—whether a firm defaults on a loan (Y=1) or not (Y=0), whether a customer churns (Y=1) or stays (Y=0)—OLS breaks down.

The Core Problem: Predictions Outside [0, 1]

With binary Y, probabilities must lie in the interval [0, 1]. But OLS estimates a linear model:

\[\hat{Y}_i = \hat{\beta}_0 + \hat{\beta}_1 X_i\]

This line extends to infinity. For large or small values of X, the fitted value will easily exceed 1 or fall below 0, producing nonsensical predictions.

Violated Regression Assumptions

1. Residuals Cannot Be Normal. If Y takes only values 0 or 1, then conditional on X, the residuals \(\hat{e}_i = Y_i - \hat{Y}_i\) can only take on two values: \(0 - \hat{Y}_i\) and \(1 - \hat{Y}_i\). A binomial distribution is far from normal.

2. Heteroscedasticity Is Guaranteed. The variance of Y given X is:

\[\text{Var}(Y|X) = p(1-p)\]

where \(p\) is the probability that Y = 1. This variance depends on X (through p), violating the homoscedasticity assumption. Moreover, when \(p \approx 0\) or \(p \approx 1\), variance is small; when \(p \approx 0.5\), variance is largest. OLS standard errors will be biased.

Worked Example: Firm Default Prediction

Let's fit OLS to a binary outcome and see the problems.

import numpy as np import pandas as pd import statsmodels.api as sm import matplotlib.pyplot as plt np.random.seed(42) # Generate data: X = financial ratio, Y = default (0/1) n = 200 X = np.random.uniform(0, 10, n) # True probability: p = 1 / (1 + exp(-(-2 + 0.5*X))) true_p = 1 / (1 + np.exp(-(-2 + 0.5*X))) Y = (np.random.random(n) < true_p).astype(int) # Fit OLS naively X_const = sm.add_constant(X) ols_model = sm.OLS(Y, X_const) ols_results = ols_model.fit() # Plot the OLS fitted line fig, ax = plt.subplots(figsize=(10, 6)) ax.scatter(X, Y, alpha=0.5, s=30, label='Data (Y = 0 or 1)') X_range = np.linspace(X.min(), X.max(), 100) X_range_const = sm.add_constant(X_range) ols_pred = ols_results.predict(X_range_const) ax.plot(X_range, ols_pred, 'r-', linewidth=2, label='OLS Fit') ax.axhline(y=0, color='gray', linestyle='--', alpha=0.5) ax.axhline(y=1, color='gray', linestyle='--', alpha=0.5) ax.set_ylabel('Predicted Probability') ax.set_xlabel('Financial Ratio (X)') ax.set_title('OLS on Binary Y: Predictions Fall Outside [0, 1]') ax.legend() ax.set_ylim([-0.3, 1.3) plt.tight_layout() plt.show() print("OLS predictions range:", ols_pred.min().round(3), "to", ols_pred.max().round(3))
OLS predictions range: -0.152 to 1.245

Notice: the OLS fitted line dips below 0 and climbs above 1. These are impossible probabilities. This is why we need a different approach.

8.2 — The Logistic Function and the Logit Model

Instead of a straight line, we need a curve that naturally stays within [0, 1]. The logistic (or sigmoid) function provides exactly that.

The Logistic Sigmoid

The logistic function is:

\[P(Y=1|X) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X)}}\]

Properties:

  • Maps all of \(\mathbb{R}\) to (0, 1). No matter how large or small \(\beta_0 + \beta_1 X\) is, the output is always a valid probability.
  • S-shaped (sigmoid). Gradual rise from near-zero at the lower tail to near-one at the upper tail.
  • Symmetric around 0.5: when \(\beta_0 + \beta_1 X = 0\), \(P(Y=1) = 0.5\).

Odds and Log-Odds

Odds: The odds of an event are the ratio of the probability it happens to the probability it doesn't:

\[\text{Odds} = \frac{p}{1-p}\]

Example: If \(p = 0.20\) (20% chance), then odds = \(\frac{0.20}{0.80} = 0.25\), or "1 to 4 against" the event.

Log-Odds (Logit): Taking the natural logarithm of odds linearizes the model:

\[\ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 X\]

Deriving the Logistic Model

Start with the log-odds equation:

\[\ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 X\]

Exponentiate both sides:

\[\frac{p}{1-p} = e^{\beta_0 + \beta_1 X}\]

Let \(z = \beta_0 + \beta_1 X\). Then:

\[p = (1-p) \cdot e^z\]

\[p = e^z - p e^z\]

\[p(1 + e^z) = e^z\]

\[p = \frac{e^z}{1 + e^z} = \frac{1}{1 + e^{-z}}\]

Thus:

\[P(Y=1|X) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X)}}\]

Code: The Sigmoid for Different Slopes

import numpy as np import matplotlib.pyplot as plt # Define sigmoid function def sigmoid(x, beta0, beta1): return 1 / (1 + np.exp(-(beta0 + beta1 * x))) X_range = np.linspace(-5, 5, 100) fig, ax = plt.subplots(figsize=(10, 6)) # Plot sigmoid for different beta1 values for beta1 in [0.5, 1.0, 2.0]: ax.plot(X_range, sigmoid(X_range, 0, beta1), label=f'β₁ = {beta1}', linewidth=2) ax.axhline(y=0.5, color='gray', linestyle='--', alpha=0.5) ax.set_xlabel('X') ax.set_ylabel('P(Y=1|X)') ax.set_title('Logistic Sigmoid for Different Slopes (β₀ = 0)') ax.legend() ax.grid(True, alpha=0.3) plt.tight_layout() plt.show()

Key Concept: The Logit Transformation

The logit transformation is the bridge between a linear model (in log-odds) and a probability model (in p). By taking log-odds, we linearize the relationship: \(\ln(p/(1-p)) = \beta_0 + \beta_1 X\). This is why the model is called logistic regression despite being a classification tool. The linear part is hidden inside the logit function.

8.3 — Interpreting Coefficients: Odds Ratios

\(\beta_1\) represents the change in log-odds per unit increase in X. But log-odds are hard to communicate. The odds ratio is much more interpretable.

Deriving the Odds Ratio

The log-odds model is:

\[\ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 X\]

Compare odds at \(X+1\) versus \(X\):

\[\ln(\text{odds}_{X+1}) - \ln(\text{odds}_X) = \beta_1\]

Using log properties, \(\ln(a) - \ln(b) = \ln(a/b)\):

\[\ln\left(\frac{\text{odds}_{X+1}}{\text{odds}_X}\right) = \beta_1\]

Exponentiate both sides:

\[\frac{\text{odds}_{X+1}}{\text{odds}_X} = e^{\beta_1}\]

This ratio is the odds ratio (OR). It tells us: when X increases by 1 unit, the odds are multiplied by \(e^{\beta_1}\).

Worked Example: Firm Default Model

Suppose we fit a logistic model of firm default and find \(\beta_1 = -0.357\) for "years of activity" (X = years operating). Then:

\[OR = e^{-0.357} \approx 0.70\]

Interpretation: For each additional year of activity, the odds of default are multiplied by 0.70, or equivalently, they decrease by 30%. A 10-year-old firm has odds of default that are \(0.70^{10} \approx 0.028\) times those of a 1-year-old firm (holding other variables constant).

Critical: Odds Ratio ≠ Probability Ratio

A common mistake is to confuse odds ratio with probability ratio. The same OR produces different changes in probability depending on the starting point.

Example: OR = 0.7 for three starting probabilities:

Starting at p = 0.50: odds = 1.00 With OR = 0.7: odds = 0.70 → p = 0.70/(1+0.70) = 0.41 Change in p: -0.09 (9 percentage points) Starting at p = 0.20: odds = 0.25 With OR = 0.7: odds = 0.175 → p = 0.175/(1+0.175) = 0.149 Change in p: -0.051 (5.1 percentage points) Starting at p = 0.05: odds = 0.0526 With OR = 0.7: odds = 0.0368 → p = 0.0368/(1+0.0368) = 0.036 Change in p: -0.014 (1.4 percentage points)

Same OR, very different impacts on probability! This is why reporting only odds ratios can be misleading.

Marginal Effects

A more interpretable metric is the marginal effect: the derivative of the probability with respect to X:

\[\frac{\partial p}{\partial X} = \beta_1 \cdot p(1-p)\]

This depends on the baseline probability p. Usually, we evaluate marginal effects at the mean of X (or at specific percentiles). The result is the percentage-point change in probability for a one-unit increase in X, at that specific point.

Code: Odds Ratios and Marginal Effects

import numpy as np import pandas as pd import statsmodels.api as sm # Generate data: firm default ~ years_active np.random.seed(42) n = 200 X = np.random.uniform(1, 20, n) true_p = 1 / (1 + np.exp(-(-2 + 0.5*X))) Y = (np.random.random(n) < true_p).astype(int) # Fit logistic regression X_const = sm.add_constant(X) logit_model = sm.Logit(Y, X_const) logit_results = logit_model.fit(disp=0) # Extract coefficients and compute OR beta = logit_results.params[1] OR = np.exp(beta) print(f"Coefficient β₁: {beta:.4f}") print(f"Odds Ratio: {OR:.4f}") print(f"Interpretation: Each additional year multiplies odds by {OR:.2f}") # Compute marginal effect at mean X X_mean = X.mean() p_mean = logit_results.predict([1, X_mean])[0] marginal_effect = beta * p_mean * (1 - p_mean) print(f"\nAt mean X = {X_mean:.2f}:") print(f"Predicted probability: {p_mean:.4f}") print(f"Marginal effect: {marginal_effect:.4f}") print(f"Interpretation: Each additional year increases probability by {marginal_effect*100:.2f} pp")

Key Concept: When to Use What

Odds ratios are useful for comparing the relative importance of predictors and for communicating results to technical audiences. For the actual impact on probability, use marginal effects or compute predicted probabilities at specific X values. In applied work, many practitioners report odds ratios alongside predicted probabilities at key reference points (e.g., at the 25th, 50th, and 75th percentiles).

8.4 — Estimation: Maximum Likelihood

Logistic regression coefficients are estimated using Maximum Likelihood Estimation (MLE), not OLS. Here's why and how.

Why Not OLS?

OLS minimizes the sum of squared residuals, which assumes:

  • Errors are normally distributed.
  • Variance is constant (homoscedasticity).

Neither holds for binary data. Instead, we use the likelihood function, which directly models the probability of observing the data we actually saw.

The Likelihood Function

For each observation i, the probability of observing \(Y_i\) given \(X_i\) is:

\[P(Y_i | X_i) = p_i^{Y_i} \cdot (1-p_i)^{1-Y_i}\]

where \(p_i = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X_i)}}\) is the predicted probability.

The likelihood (probability of all the data) is the product:

\[L = \prod_{i=1}^{n} p_i^{Y_i} \cdot (1-p_i)^{1-Y_i}\]

The log-likelihood (easier to work with) is:

\[\ell = \sum_{i=1}^{n} \left[ Y_i \ln(p_i) + (1-Y_i) \ln(1-p_i) \right]\]

MLE finds the values of \(\beta_0\) and \(\beta_1\) that maximize \(\ell\). There is no closed-form solution, so numerical algorithms (like Newton-Raphson) are used.

Worked Example: Firm Default with Three Predictors

Let's generate data for firm default as a function of debt-to-income ratio, years active, and log revenue, then fit the model.

import numpy as np import pandas as pd import statsmodels.api as sm np.random.seed(123) # Generate 300 firms n = 300 data = pd.DataFrame({ 'debt_ratio': np.random.uniform(0.1, 2.0, n), 'years_active': np.random.uniform(1, 30, n), 'log_revenue': np.random.uniform(10, 18, n) }) # True model: default probability depends on these factors z = -3 + 1.2*data['debt_ratio'] - 0.08*data['years_active'] + 0.15*data['log_revenue'] data['default'] = (np.random.random(n) < 1/(1+np.exp(-z))).astype(int) # Fit logistic model X = sm.add_constant(data[['debt_ratio', 'years_active', 'log_revenue']]) model = sm.Logit(data['default'], X) results = model.fit() print(results.summary2())

8.5 — Goodness of Fit

Unlike linear regression, logistic regression has no traditional R-squared. Instead, we use pseudo-R² measures and information criteria.

McFadden's Pseudo-R²

A common measure is:

\[R^2_{\text{McFadden}} = 1 - \frac{\ell_{\text{model}}}{\ell_{\text{null}}}\]

where \(\ell_{\text{model}}\) is the log-likelihood of your fitted model and \(\ell_{\text{null}}\) is the log-likelihood of a model with only an intercept (predicting the overall default rate for everyone).

Values between 0.2 and 0.4 are often considered good. Higher values suggest the model explains more variation in the outcome.

Log-Likelihood Ratio Test

To compare nested models (e.g., a model with X and one without), use the LR test statistic:

\[LR = 2(\ell_{\text{full}} - \ell_{\text{reduced}})\]

Under the null hypothesis that the extra variables don't matter, LR follows a chi-squared distribution with degrees of freedom equal to the number of extra predictors.

AIC and BIC

For comparing non-nested models, use AIC or BIC (introduced in Module 06):

\[AIC = 2k - 2\ell_{\text{model}}\]

\[BIC = k \ln(n) - 2\ell_{\text{model}}\]

Lower values are better. BIC penalizes model complexity more heavily, favoring simpler models.

Code: Extracting Model Fit Metrics

# From the previous fit result print("Pseudo R² (McFadden):", results.prsquared.round(4)) print("Log-Likelihood:", results.llf.round(2)) print("AIC:", results.aic.round(2)) print("BIC:", results.bic.round(2))

8.6 — Classification: From Probabilities to Decisions

Logistic regression produces predicted probabilities. To make yes/no decisions, we need a threshold.

Threshold Selection

The default threshold is 0.5: if \(\hat{p}_i \geq 0.5\), predict Y=1; otherwise predict Y=0. But the optimal threshold depends on context.

Conservative (low threshold, ~0.3): Predict Y=1 for more observations. Useful when missing positives is costly (e.g., detecting fraud). Maximizes recall.

Aggressive (high threshold, ~0.7): Predict Y=1 only for high-confidence cases. Useful when false positives are costly (e.g., spam filters). Maximizes precision.

Confusion Matrix and Classification Metrics

For a chosen threshold, we can construct a confusion matrix:

Predicted Y=1 Y=0 Actual Y=1 TP FN Y=0 FP TN TP = True Positives (correctly predicted 1) FP = False Positives (incorrectly predicted 1) TN = True Negatives (correctly predicted 0) FN = False Negatives (incorrectly predicted 1)

Accuracy: \(\frac{TP + TN}{N}\) — overall correctness. Can be misleading if classes are imbalanced.

Precision: \(\frac{TP}{TP + FP}\) — of predicted positives, how many are correct? "Don't cry wolf too often."

Recall (Sensitivity): \(\frac{TP}{TP + FN}\) — of actual positives, how many did we catch? "Don't miss the real wolves."

Specificity: \(\frac{TN}{TN + FP}\) — of actual negatives, how many did we correctly identify as negative?

F1-Score: Harmonic mean of precision and recall: \(\frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}\). Balances both metrics.

Code: Confusion Matrix and Metrics at Different Thresholds

from sklearn.metrics import confusion_matrix, accuracy_score, precision_score, recall_score # Get predicted probabilities pred_probs = results.predict(X) # Try different thresholds for threshold in [0.3, 0.5, 0.7]: pred_binary = (pred_probs >= threshold).astype(int) cm = confusion_matrix(data['default'], pred_binary) acc = accuracy_score(data['default'], pred_binary) prec = precision_score(data['default'], pred_binary) rec = recall_score(data['default'], pred_binary) print(f"\nThreshold = {threshold}") print(f"Confusion Matrix:\n{cm}") print(f"Accuracy: {acc:.3f}, Precision: {prec:.3f}, Recall: {rec:.3f}")

Key Concept: Threshold Is a Business Decision

There is no universally correct threshold. The choice reflects the relative cost of false positives vs false negatives. In credit risk, a bank might tolerate more false positives (lending to marginal borrowers) to avoid false negatives (rejecting good borrowers). In medical screening, you might tolerate more false positives to catch all true cases. Always anchor threshold choice to business context.

8.7 — The ROC Curve and AUC

Rather than choose one threshold, the ROC (Receiver Operating Characteristic) curve shows how classification performance varies across all possible thresholds.

Constructing the ROC Curve

For each threshold from 0 to 1, compute:

  • Sensitivity (Recall): \(\frac{TP}{TP + FN}\) — true positive rate on the vertical axis.
  • 1 − Specificity: \(\frac{FP}{FP + TN}\) — false positive rate on the horizontal axis.

Plot these points for all thresholds. A perfect classifier hugs the top-left corner (high sensitivity, low false positive rate). A random classifier follows the diagonal (sensitivity = false positive rate).

Area Under Curve (AUC)

The AUC summarizes the entire ROC curve into a single number (0 to 1):

  • AUC = 0.5: Random guessing.
  • AUC = 0.7–0.8: Acceptable discrimination.
  • AUC = 0.8–0.9: Good discrimination.
  • AUC > 0.9: Excellent discrimination.

Interpretation: AUC is the probability that, if you pick a random positive case and a random negative case, the model assigns a higher predicted probability to the positive case.

Code: Plotting ROC Curve and Computing AUC

from sklearn.metrics import roc_curve, auc import matplotlib.pyplot as plt # Compute ROC curve fpr, tpr, thresholds = roc_curve(data['default'], pred_probs) roc_auc = auc(fpr, tpr) # Plot fig, ax = plt.subplots(figsize=(8, 6)) ax.plot(fpr, tpr, color='b', linewidth=2, label=f'ROC Curve (AUC = {roc_auc:.3f})') ax.plot([0, 1], [0, 1], color='gray', linestyle='--', label='Random Classifier') ax.set_xlabel('False Positive Rate') ax.set_ylabel('True Positive Rate (Recall)') ax.set_title('ROC Curve') ax.legend() ax.grid(True, alpha=0.3) plt.tight_layout() plt.show()

Key Concept: AUC and Discrimination

AUC measures the model's ability to discriminate between classes across ALL thresholds. It answers the question: "If I pick a random positive case and a random negative case, what's the probability the model assigns a higher score to the positive?" A higher AUC means better discrimination. Unlike accuracy, AUC is robust to class imbalance.

8.8 — Multiple Logistic Regression and Practical Considerations

Extending to multiple predictors is straightforward: just add more X variables.

The Model

\[P(Y=1|X_1, X_2, \ldots, X_k) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_k X_k)}}\]

Each \(\beta_j\) is still interpreted as the change in log-odds per unit change in \(X_j\), holding all other variables constant (ceteris paribus). The corresponding odds ratio is \(e^{\beta_j}\).

Categorical Predictors

Use dummy variables (one-hot encoding), dropping one category as the reference. The coefficients for other categories represent deviations from the reference.

Multicollinearity

Check for multicollinearity using the Variance Inflation Factor (VIF), just as in linear regression (Module 06). High VIF indicates that a predictor is highly collinear with others, inflating its standard error.

Connection to Chi-Squared Tests (Module 04)

Chi-squared tests (Module 04) test whether two categorical variables are independent. Logistic regression extends this idea: it models the probability of a categorical outcome (Y = 0 or 1) as a function of multiple predictors, some categorical, some continuous. Logistic regression is the multivariate extension of chi-squared tests.

Beyond Binary: Multinomial and Ordinal Logistic Regression

When Y has more than two categories:

  • Unordered categories (e.g., transportation mode: car, bus, train): Use multinomial logistic regression. Fit k-1 equations, one for each non-reference category.
  • Ordered categories (e.g., credit rating: poor, fair, good, excellent): Use ordinal logistic regression (proportional odds model), which respects the ordering.

Worked Example: Adding Industry Sector

# Add industry sector to our firm default data data['industry'] = np.random.choice(['Manufacturing', 'Services', 'Tech'], n) # Create dummies (drop first) industry_dummies = pd.get_dummies(data['industry'], drop_first=True) # Fit model with industry dummies X_with_industry = sm.add_constant(pd.concat([ data[['debt_ratio', 'years_active', 'log_revenue']], industry_dummies ], axis=1)) model_industry = sm.Logit(data['default'], X_with_industry) results_industry = model_industry.fit(disp=0) print(results_industry.summary2())

Interpretation: The coefficients on Services and Tech are relative to Manufacturing (the reference category). If the Services coefficient is negative, firms in Services have lower default odds compared to Manufacturing, holding financial ratios constant.

Key Takeaways

  • OLS fails for binary outcomes. Predictions fall outside [0, 1], residuals violate normality and homoscedasticity. Use logistic regression instead.
  • The logistic sigmoid maps all inputs to (0, 1). It's an S-shaped curve that naturally produces valid probabilities.
  • Interpret via odds ratios. \(e^{\beta_j}\) is the factor by which odds multiply for a one-unit increase in \(X_j\). But remember: odds ratio ≠ probability ratio.
  • Use marginal effects or predicted probabilities for applied interpretation. They directly show the impact on probability, conditional on a baseline level.
  • MLE finds coefficients that maximize the likelihood of observing the actual data. No closed-form solution; numerical algorithms required. But standard software handles this.
  • Pseudo-R², AIC, and BIC assess model fit. Use them to compare models. McFadden's R² of 0.2–0.4 is often considered good.
  • Threshold determines classification. Low thresholds maximize recall; high thresholds maximize precision. Choose based on the cost of false positives vs false negatives.
  • AUC summarizes performance across all thresholds. It's robust to class imbalance and directly interpretable: probability that a random positive case ranks higher than a random negative case.
  • Logistic regression extends to multiple predictors, categorical variables, and beyond-binary outcomes. Same principles; just more Xs. For ordered outcomes, use ordinal logistic regression.
